⚡ 元件系統說明
元件插入武器槽位（觸元件槽 / 應元件槽），組合成「觸發條件 → 應用效果」的效果對。
觸元件（Condition）定義何時生效，應元件（Function）定義發生什麼效果。
模擬器計算範圍僅涵蓋 6 類效果：增傷 / 增彈 / 增倍率 / 破甲 / AP增傷 / 軀幹增傷。
篩選功能
元件類型切換（觸元件 / 應元件 / 全部） 依稀有度（EX / S / A / B，對應 API quality SSR/SR/R/N；標籤配色：EX=橙、S=金、A=紫、B=藍） W 類型（W型 / 一般，傷害計算機相關） 觸發條件類型（觸元件用：雙持 / 單持 / 首次攻擊 / AP消耗 / 攻擊軀幹 / 每次攻擊） 效果類型（應元件用：增傷 / 增彈 / 增倍率 / 破甲 / AP增傷 / 軀幹增傷） 關鍵字搜尋（元件名稱 / 描述文字）

⚙️

元件卡片

元件圖示（雙層疊合：框架圖 + 技能圖示）、名稱、稀有度標籤、componentType 標籤（觸元件 / 應元件）、W 類型標籤（W / 一般）。
觸元件顯示 conditionType 觸發標籤（雙持 / 單持 / 首次攻擊 / AP消耗 / 攻擊軀幹 / 每次攻擊）；
應元件顯示 effectType 效果標籤（增傷 / 增彈 / 增倍率 / 破甲 / AP增傷 / 軀幹增傷）。

🖼️

元件圖示雙層結構

圖示由兩層圖片疊合（CSS position: relative/absolute）：
① 框架圖（底層）：路徑優先取 outerFrameLocal 欄位（由 patch 腳本寫入），若空則自動推算 /images/components/OuterFrame/statetype_{componentType}{_W}.png。共四種： statetype_Condition.png、statetype_Condition_W.png、statetype_Function.png、statetype_Function_W.png，依 componentType × componentsWType 決定。 框架圖佔容器 80%，置中顯示。
② 技能圖示（上層）：取 iconLocal 欄位（擷取腳本下載後的本地路徑，如 /images/components/Icon_skill_passive_5223.png）。 圖示佔容器 48%，以 translate(-51%, -52%) rotate(16deg) 略微偏移並旋轉以貼合框架設計。
技能圖示不存在時（onError）隱藏；框架圖固定存在，不需 fallback。

📝

效果描述

觸元件：顯示 condition 觸發條件文字（如「同時使用兩把武器攻擊時」）。
應元件：顯示 description 完整效果描述文字。
描述中的數字與 % 符號以醒目色標示（參考 highlightNumbers() 實作）。

🔗

武器限制標籤

allowedWeaponTypes 列出允許裝備的武器大類（射擊 / 格鬥 / 突擊 / 戰術）。
若非全類型皆可裝備，以標籤列形式顯示限制條件。
W 型元件（componentsWType = "W"）額外標示「僅限雙手 / 背後武器」提示。

📊

元件子類型標示

moduleSubtype（1–11）以小標籤顯示對應中文分類，如「命中相關」、「攻擊結果」等。
可作為次要篩選維度，供進階用戶篩選特定類別的元件。

📱

行動裝置底部彈層

行動裝置點擊卡片時，以 BottomSheet 展示完整元件詳細資訊（觸發條件、效果類型、武器限制、probabilityLevel 等）。
參考模組圖鑑的 BottomSheet 實作模式。

📐 版面 格線卡片（1 欄 / 平板 2 欄 / 寬螢幕 3 欄），與模組圖鑑相同佈局。
頁面頂部設置類型切換標籤（觸元件 / 應元件 / 全部），作為主要分類入口；切換後篩選條件（conditionType / effectType）即時替換。
篩選區域固定於內容區頂部，不含側欄。

實作注意事項
元件圖示渲染（對應 AdminPage.tsx ComponentAdmin 實作）
· 外框圖路徑：優先讀取 comp.outerFrameLocal（由 patch-component-outerframe.mjs 寫入）；若空白則自動推算：
 /images/components/OuterFrame/statetype_{componentType}{_W}.png
 四種變體：statetype_Condition.png / statetype_Condition_W.png / statetype_Function.png / statetype_Function_W.png
· 技能圖示路徑：取 comp.iconLocal（完整本地路徑，如 /images/components/Icon_skill_passive_5223.png）。icon 欄位僅為 key，不直接用於 <img src>
· CSS 疊合：容器 position: relative；外框圖 absolute，width/height: 80%，translate(-50%, -50%) 置中；技能圖示 absolute，width/height: 48%，translate(-51%, -52%) rotate(16deg)（略微偏移旋轉以貼合框架外形）
· 圖示不存在時（onError）設 display:none，避免破圖；外框圖固定存在，不需 fallback
篩選切換行為
· 切換 componentType（觸元件 / 應元件）時，conditionType 篩選器與 effectType 篩選器互斥顯示
· 選「全部」時，conditionType 與 effectType 篩選器同時顯示，各自獨立過濾
· W 類型篩選器在觸元件 / 應元件 / 全部 模式下均顯示
probabilityLevel 說明
· 對應 WIKI ProbabilityLevel 欄位，代表元件等級
· 圖鑑頁可顯示等級數字，未來需確認是否影響效果數值（目前僅作標示用）
