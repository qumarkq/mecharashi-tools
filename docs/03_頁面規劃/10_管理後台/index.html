管理後台ADMIN PAGE · /admin

Firestore 遊戲資料直接編輯、攻略管理、公告管理（管理員限定）

頁面規格
管理後台 /admin
路由/admin
開發階段Phase 2（部分提前實作）；完整 Phase 6
登入需求目前：無（本地開發用）。Phase 6：Firebase Auth admin custom claim
資料來源目前：public/data/*.json（fetchData 讀取，匯出 JSON 覆蓋）。Phase 6：Firestore 直接讀寫
✅ 目前已實作（Firestore 模式） 載入 Firestore modules / mechs 集合 → 在頁面內編輯 → 儲存後直接寫回 Firestore，無需匯出 JSON 或 Git commit。

模組等級編輯（levels）： 模組編輯面板內新增「模組等級 levels」區塊，可新增 / 移除各等級，並對每個等級個別設定 level、description 及全部數值欄位（基礎屬性 + 武器專屬增傷）。 各等級預設收合顯示，點擊展開編輯，第一個等級預設展開。

⚠️ Phase 6 安全性目標 管理後台路由需驗證 Firebase Auth 的 admin custom claim，未授權用戶自動重導到首頁。 Firestore Security Rules 亦需同步設定寫入權限，禁止非管理員寫入遊戲資料 Collections。

功能模組
🗄️

遊戲資料管理

各 Collection（pilots/mechs/weapons 等）的列表 + 新增/編輯/刪除。Firestore 直接讀寫，無需 CI/CD。

目前：模組管理（modules.json）已實作，含搜尋/篩選/編輯/匯出 JSON。品質與部位欄位由 Enum 驅動。

🧩

模組-機甲關聯編輯

（Phase 2 提前）管理副模組與機甲的關聯關係（module4Id / module8Id / moduleFixedIds）。

目前：機甲管理（Firestore mechs）已實作，含模組 ID 下拉選單、未對應模組警示（⚠ 紅框提示）。

模組等級（levels）：模組編輯面板支援各 LV 能力值的新增、移除與編輯，涵蓋基礎屬性及武器專屬增傷。

✏️

攻略文章管理

Markdown 編輯器撰寫攻略，Firebase Storage 上傳截圖，Firestore 儲存文章。

📢

公告 / 改版資訊

透過後台或 Firestore 直接管理公告欄與改版時間軸內容。

👥

用戶管理

查看用戶列表，授予/撤銷管理員 custom claim。（只讀為主）
