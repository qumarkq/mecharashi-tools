03 — Damage Formula
傷害公式分析
根據試算表與 NGA 社群研究整理。
核心傷害公式 最終傷害 = (總攻擊 - 敵方裝甲) × 增傷係數 × 爆擊係數 × 最終傷害係數
總攻擊展開 總攻擊 = (武器基礎攻擊力 + 固定改裝攻擊 + 浮動改裝攻擊) × 火力係數 + 科研火力 + 改造火力) × 五維係數 × 敵等級係數 × 駕駛等級係數 · 火力係數：每 3650 點火力 = 武器基礎攻擊力 ×100%（70EX 等級基準）
· 五維係數：近似三次方曲線，中心點約 8200-8400
· 敵等級係數：三次方曲線，Lv0 無衰減，Lv55 衰減 12.5%，Lv71 衰減 25%
· 固定改裝攻擊：基礎改裝提供的固定攻擊加成
· 浮動改裝攻擊：高級改裝的隨機攻擊加成（含條件觸發加成）
傷害期望值 期望值 = 基礎傷害 × (1 + 增傷%) × [ 爆率% × (1 + 爆傷%) + (1 - 爆率%) ] × 命中率% × 連擊數 × 技能倍率
增傷%來源匯總 (v1.2 更新) 增傷% = 機甲模組(四模/八模/固定) + 應元件(增傷型) + 背包技能 + 武器技能(攜帶即生效/裝備中生效)
        + 武器改裝(浮動改裝條件觸發) + 機師技能 + 天賦 + 神經驅動 + 機師個別科研 + 全域科研(增傷項) 爆率%、爆傷%、命中% 亦各自匯總上述所有來源中對應的加成；子彈數/倍率/破甲/AP增傷/軀幹增傷由元件額外計算
⚠️ 待確認 重甲單位 <1% 傷害偏差原因未知。五維係數精確曲線參數、敵等級 Lv71+ 外推準確度待驗證。建議模擬器提供進階選項讓使用者自行微調。

▎元件插槽
triggerSlotsnumber觸元件插槽數量 (不同武器不同，如 3)
effectSlotsnumber應元件插槽數量 (不同武器不同，如 4)
▎固定改裝（基礎改裝）
fixedModobject固定改裝方案（依武器等級解鎖，效果固定）↓
 ├ planNamestring改進方案名稱 (如 "機槍VIII")
 ├ maxLevelnumber可改裝等級上限 (如 70)
 └ effects[]array固定效果列表 { stat, value }
stat: "attack"/"crit"/"accuracy"
如 { stat: "attack", value: 107 }
▎浮動改裝（高級改裝）
floatingModobject浮動改裝方案（效果隨機，有範圍）↓
 ├ planNamestring進階方案名稱 (如 "機槍IV")
 ├ slotsnumber可裝填效果欄位數 (如 3)
 └ possibleEffects[]array所有可能獲得的效果 ↓
  ├ statstring效果類型：attack / crit / accuracy / firepower
  ├ conditionstring?觸發條件 (null=無條件)
"targetHP<50%" / "targetHP>50%"
"vsLight" / "vsMedium" / "vsHeavy"
  ├ minnumber數值下限 (如 53)
  └ maxnumber數值上限 (如 160)
▎武器技能
skills[]array武器技能列表 ↓
 ├ namestring技能名稱 (如 "白鶴之眺")
 ├ typestring技能類型 (被動技能/主動技能)
 ├ activationstring生效方式：
"carry" = 攜帶即生效
"equip" = 裝備中生效
"use" = 僅使用時生效
 ├ descriptionstring技能效果描述
 ├ dmg / crit / critDmg / accnumber可量化的加成數值
 └ enhancesTalentstring?加強的特定機師天賦名稱 (專武專用)
💡 固定改裝 vs 浮動改裝 固定改裝（基礎改裝）：效果確定，達到等級即解鎖。浮動改裝（高級改裝）：效果從可能池中隨機獲得，數值有範圍。模擬器中由使用者手動輸入自己已改裝的浮動效果。管理者在 Google Sheets 中針對各武器種類設定浮動範圍。

📁 components.json — 元件資料庫 UPDATED
🔄 v1.2 修正：元件系統完整建模 元件分為「觸元件」(觸發條件) 和「應元件」(觸發後效果)。每個元件有等級與概率屬性。模擬器僅計算以下效果類型：增傷、增加子彈、增加倍率、破甲、每消耗N點AP增傷、對軀幹增傷。
欄位型別說明
▎觸元件（觸發條件）
id / namestring元件名稱 (如 "觸元件-共力")
slot"trigger"固定為 "trigger"，表示觸元件
levelnumber元件等級 (如 Lv.4)
probabilitystring觸發概率類型："概率" / "必定"
conditionstring觸發條件描述 (如 "同時使用兩把武器攻擊時")
conditionTypestring條件分類：
"dualWield" = 同時使用兩把武器
"singleWield" = 使用單把武器
"firstAttack" = 首次攻擊
"apCost" = 每消耗N點AP
"targetTorso" = 攻擊軀幹
"always" = 每次攻擊
conditionValuenumber?條件參數 (如 apCost 類型填 AP 數值)
▎應元件（觸發後效果）
id / namestring元件名稱 (如 "應元件-強擊")
slot"effect"固定為 "effect"，表示應元件
levelnumber元件等級 (如 Lv.5)
probabilitystring觸發概率類型："概率" / "必定"
effectTypestring效果分類（僅收錄模擬器需要的）：
"dmgBoost" = 增傷 (如 傷害提升20%)
"bulletAdd" = 增加子彈 (如 發射子彈數+1)
"multiplierBoost" = 增加倍率 (如 攻擊倍率提升0.1)
"armorBreak" = 破甲 (降低敵方裝甲)
"apDmgBoost" = 每消耗N點AP增傷
"torsoDmgBoost" = 對軀幹增傷
valuenumber效果數值 (如 20 代表 20%，或 1 代表 +1 子彈)
descriptionstring完整效果描述文字
raritystring稀有度 (金/紫/藍)
⚡ 元件模擬範圍限定 遊戲中元件效果種類繁多，但本站模擬器僅計算以下 6 類對傷害有直接影響的效果：
① 增傷（傷害提升%）② 增加子彈（發射子彈數+N）③ 增加倍率（攻擊倍率提升）④ 破甲（降低敵方裝甲值）⑤ 每消耗N點AP增傷（與技能AP消耗關聯）⑥ 對軀幹增傷
其他效果（如移動力、防禦相關）不在傷害模擬範圍內。

06 — Simulator Flow
配裝模擬器操作流程
v1.2 更新：加入武器改裝、元件插槽（觸/應元件）、背包限制、科研選擇的完整流程。

模擬器步驟流程

STEP 1
選擇機師
自動帶入六維、技能、天賦、神經驅動、駕駛等級
STEP 2
選擇機甲
自動帶入火力、模組、裝甲類型。根據駕駛許可篩選可選機甲
STEP 3
選擇武器
根據機甲出力篩選（重量限制），顯示所有技能與生效方式。專武天賦加強自動標記
STEP 4
選擇背包
根據機甲裝甲類型篩選（如「僅限輕型」），根據剩餘出力篩選重量
STEP 5
配置機師個別科研
機師/機甲/武裝特質各 I/II/III 欄位，有互斥提示
STEP 6
配置武器改裝
固定改裝自動帶入；浮動改裝由使用者選擇已解鎖的效果（含條件觸發效果）
STEP 7
配置元件（觸/應）
根據武器的觸元件/應元件插槽數量，選擇觸發條件與效果。僅計算：增傷/增彈/增倍率/破甲/AP增傷/軀幹增傷
STEP 8
設定敵方 + 戰鬥參數
敵方機師/機甲、技能倍率、連擊數、是否最佳射程、AP 消耗量
RESULT
即時計算結果
增傷%/爆率%/爆傷%/命中%/期望值，支援 A/B 對比與匯出圖片
