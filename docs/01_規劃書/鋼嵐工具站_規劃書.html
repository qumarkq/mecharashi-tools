01 — Architecture
系統架構概覽
前端部署於 GitHub Pages，Firebase Firestore 統一管理所有遊戲資料與用戶資料。 → 完整架構文件
🔄 v2.0 架構重大調整：靜態 JSON → Firebase Firestore 舊架構：Google Sheets → GitHub Actions → Static JSON → 前端讀取。
新架構：管理後台 → Firestore 遊戲資料 → 前端即時讀取。
優點：資料即時更新、複雜關聯可直接透過 Document ID 引用、型別枚舉化消除不一致。
