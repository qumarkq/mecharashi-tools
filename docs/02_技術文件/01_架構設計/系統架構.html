v2.0 — Firebase Firestore
概覽
架構說明
前端部署於 GitHub Pages；Firebase Firestore 統一管理所有遊戲資料與用戶資料。管理者透過網站後台直接讀寫 Firestore，無需 CI/CD。
🔄 v2.0 架構重大調整：靜態 JSON → Firebase Firestore 舊架構：Google Sheets → GitHub Actions → Static JSON → 前端讀取。
新架構：管理後台 → Firestore 遊戲資料 → 前端即時讀取。
優點：資料即時更新、複雜關聯（模組↔機甲、技能↔BUFF）可直接透過 Document ID 引用、型別枚舉化消除不一致。

遊戲資料讀取

前端透過 Firebase SDK 直接查詢 Firestore Collections（pilots / mechs / weapons 等），使用 React Hooks 封裝資料讀取邏輯，支援快取策略。

👤

用戶資料讀取

Firebase Auth 取得 UID 後，讀取 users/{uid}/profile 與 users/{uid}/builds。未登入用戶的科研設定暫存於 localStorage。

🛠️

管理後台寫入

管理員登入後，透過 AdminPage 直接對 Firestore 進行 CRUD 操作。角色驗證透過 Firebase Auth custom claims 實作。

GitHub Pages

前端靜態資源部署至 GitHub Pages。推送 main 分支自動觸發 GitHub Actions 建置並部署。

🔥

Firebase

Firestore、Auth、Storage 使用 Firebase Spark（免費方案）起步，流量成長後可升級至 Blaze 計費方案。
