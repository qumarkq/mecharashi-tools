01 — Schema
mechs Collection 欄位定義
每份 Document 均自動附帶 Firestore 系統欄位（_id、createdAt、updatedAt），以下僅列業務欄位。
🗃️ mechs — 機甲資料
欄位型別說明
idstring唯一識別碼（如 "mech_都卜勒"）
namestring機甲名稱
rarityItemRarity稀有度 Enum
armorTypeArmorType裝甲類型 Enum：輕型 / 中甲 / 重型
firepowernumber整體火力值（軀幹滿級）
armornumber整體裝甲值
evasionnumber閃避值
mobilitynumber移動力
weightnumber總重量
outputnumber總出力
partsobject部件詳細資料（見下方）
module4Idstring副模組 ID（4模組位置）→ 引用 modules collection
module8Idstring副模組 ID（8模組位置）→ 引用 modules collection
moduleFixedIds[]string[]固定副模組 ID 列表 → 引用 modules collection
▎parts.{position} 結構（torso / leftArm / rightArm / legs）
 ├ durablenumber耐久值
 ├ armornumber護甲值
 ├ firepowernumber火力值
 ├ weightnumber部件重量
 ├ interfacestring接口類型（如 "Ⅱ型接口"）
 ├ hitnumber?命中值（僅左右臂）
 ├ dodgenumber?閃避值（僅腿部）
 ├ movenumber?移動力（僅腿部）
 ├ outputnumber?出力（僅軀幹）
 ├ antiRiotnumber?抗暴（僅軀幹）
 └ iconstring?部件圖示路徑
02 — Relations
關聯說明
🔗 模組引用 mechs document 透過 module4Id、module8Id、moduleFixedIds[] 引用 modules collection 中的副模組文件（slot = BUILT_IN 的模組）。 查詢特定機甲的副模組時，可使用 modules collection 的複合索引 boundMechId + boundPart 反向查詢。
