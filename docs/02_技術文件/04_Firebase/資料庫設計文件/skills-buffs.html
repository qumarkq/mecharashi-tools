01 — skills
skills Collection 欄位定義
📌 用途說明 機師的 talents / skills / neuralDrive 以嵌入方式存於 pilots 文件中（1:1 關係）。
skills collection 存放「可跨實體引用」的共用技能定義，例如：多個武器共享的技能效果模板、或用於 BUFF 系統的觸發技能。
初期可作為技能索引使用，後續依需求擴充引用關係。
🗃️ skills — 技能庫
欄位型別說明
idstring唯一識別碼（如 "skill_001"）
namestring技能名稱
typeSkillType技能類型 Enum：被動技能 / 主動技能
descriptionstring效果描述
iconstring?圖示路徑
dmg / crit / critDmg / accnumber可量化加成數值
tags[]string[]分類標籤（如 ["pilot", "talent"]、["weapon", "passive"]）
relatedEntityIdstring?關聯實體 ID（如機師 ID 或武器 ID）
buffIds[]string[]?此技能施加的 BUFF ID 列表 → 引用 buffs collection
複合索引
欄位組合用途
tags ARRAY_CONTAINS, type ASC依標籤搜尋技能
02 — buffs
buffs Collection 欄位定義
🆕 v1.0 新增：BUFF Collection（初始版本） 初始版本僅定義 BUFF 名稱與描述，作為佔位設計。後續版本將擴充：持續時間、疊加規則、效果欄位（effects[]）、觸發來源等。 設計原則：任何技能、模組、武器皆可透過 buffIds[] 陣列引用此 Collection 的 BUFF 定義。
🗃️ buffs — BUFF 定義
欄位型別說明
idstring唯一識別碼（如 "buff_001_增傷"）
namestringBUFF 名稱（如 "護甲強化"、"增傷Lv3"）
descriptionstringBUFF 效果文字描述
iconstring?BUFF 圖示路徑（可留空）
▎預留欄位（後續版本擴充）
durationnumber?持續回合數（null = 永久）
maxStacknumber?最大疊加層數（1 = 不可疊加）
effects[]array?具體效果列表 { stat, value, condition } — 待設計
buffTypestring?BUFF 類型分類（增益 / 減益 / 控制 / 護盾）— 待枚舉
