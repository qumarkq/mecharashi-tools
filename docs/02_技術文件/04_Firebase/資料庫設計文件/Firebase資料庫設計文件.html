01 — Design Principles
設計原則與架構說明
從靜態 JSON 檔案遷移至 Firebase Firestore，集中管理所有遊戲資料，解決複雜關聯性與維護困難的問題。
🔗

關聯管理

模組 ↔ 機甲、技能 ↔ 機師、BUFF ↔ 技能等跨實體關聯，透過 Firestore Document ID 引用，取代 JSON 中的字串 ID 難以驗證的問題。

📐

型別枚舉化

模組槽位、職業、裝甲類型、稀有度等分類欄位，統一定義為 TypeScript Enum，前端與後端共用同一份定義，避免拼寫錯誤與不一致。

🛡️

安全規則分層

遊戲資料 Collections：所有人可讀、僅管理員可寫。用戶 Collection：僅本人可讀寫。Firebase Security Rules 強制執行。

⚡

查詢優化

針對常用篩選場景（依職業篩機師、依槽位篩模組、依機甲篩副模組）預設複合索引，確保大資料量下的查詢效能。

🔄

可擴充設計

BUFF Collection 初始僅含名稱與描述，欄位設計為可向後兼容擴充。技能 Collection 預留 tags、relatedEntities 等泛化欄位。

📋

Admin 管理介面

遊戲資料改由網站管理後台（AdminPage）直接讀寫 Firestore，取代 Google Sheets + GitHub Actions 的間接流程。

03 — Collections Overview
Firestore Collections 清單
共 13 個 Collections，分為「遊戲資料」（管理員維護）與「用戶資料」（用戶私有）兩大類。
pilots
GAME DATA

機師完整資料（六維、技能、天賦、神經驅動、AP）

mechs
GAME DATA

機甲完整資料（部件、火力、出力、模組引用）

modules
GAME DATA

所有模組（4模/8模/通用/副模組，含各等級效果）

weapons
GAME DATA

武器資料（技能、改裝、元件插槽）

backpacks
GAME DATA

背包裝備（技能效果、機甲類型限制）

components
GAME DATA

觸元件 + 應元件（等級、觸發條件、效果）

skills
GAME DATA

技能庫（可跨實體引用的共用技能定義）

buffs
GAME DATA · NEW

BUFF 定義（初始：名稱 + 描述，後續擴充效果欄位）

pilotResearch
GAME DATA

各機師個別科研選項（三種特質 × I/II/III 欄位）

globalResearch
GAME DATA

全域科研定義（機師/機甲/武器 三大類加成）

enemies
GAME DATA

敵方機師+機甲（傷害模擬器用）

users/{uid}/profile
USER DATA

用戶設定（全域科研等級、機師科研選擇、角色：USER / ADMIN / OWNER）

users/{uid}/builds/{id}
USER DATA

用戶配裝紀錄（機師+機甲+武器+背包+模組+元件）
