01 — Schema
components Collection 欄位定義
每份 Document 均自動附帶 Firestore 系統欄位（_id、createdAt、updatedAt），以下僅列業務欄位。
componentType 對應 WIKI mQuery ModuleType 欄位。
🗃️ components — 共用欄位
欄位型別說明
idstring元件識別碼
namestring元件名稱
componentTypeComponentType類型 Enum：Condition（觸元件）/ Function（應元件）；對應 WIKI ModuleType
moduleSubtypeModuleSubtype子類型 Enum（數字 1–11）；對應 WIKI ModuleSubtype
probabilityLevelnumber元件等級；對應 WIKI ProbabilityLevel 欄位
descriptionstring完整效果描述
allowedWeaponTypesWeaponType[]允許裝備的武器種類；預設全給，管理者手動維護
rarityItemRarity稀有度 Enum；API quality 映射：SSR → EX（橙）/ SR → S（金）/ R → A（紫）/ N → B（藍）；標籤配色：EX=橙、S=金、A=紫、B=藍
iconstring技能圖示 key（API IconPath 欄位，如 Icon_skill_passive_5223）；顯示時疊於框架圖（statetype_{componentType}.png）上
iconLocalstring本機技能圖示路徑（/images/components/{icon}.png）；由擷取腳本自動填入
outerFrameLocalstring本機外框圖路徑（/images/components/OuterFrame/statetype_{componentType}[_W].png）；依 componentType × componentsWType 決定，共四種；由 patch-component-outerframe.mjs 填入
componentsWTypeComponentsWTypeW 類型標記 Enum：W = W型元件（僅限 WeaponEquipSlot DUAL_HAND / BACK 的武器可裝備，傷害計算機用）；Normal = 一般元件；依元件名稱是否含「W」字樣自動推斷
▎Condition 專用欄位（componentType = Condition）
conditionTypeConditionType觸發條件類型 Enum：dualWield / singleWield / firstAttack / apCost / targetTorso / always
conditionstring觸發條件描述文字
▎Function 專用欄位（componentType = Function）
effectTypeEffectType效果類型 Enum：dmgBoost / bulletAdd / multiplierBoost / armorBreak / apDmgBoost / torsoDmgBoost

02b — 補充說明
Boss 掉落資料（本機靜態，非 Firestore）
元件 Boss 掉落資訊不存入 Firestore，以本機靜態 TypeScript 檔案維護，無需雲端查詢即可篩選。
項目說明
檔案位置src/data/bossDrops.ts
資料結構BOSS_DROPS[componentType][componentsWType][shortName] → StageDrop[]（StageDrop: { stage: number; bosses: number[] }）
圖片路徑/images/components/Stage{N}_Boss/character_{N}_{bossNum}.png（存放於 public/images/components/）
維護方式手動編輯 bossDrops.ts，新增關卡時在對應元件條目加入 { stage: N, bosses: [...] }
關卡篩選getAllStages() 動態從資料中取得所有關卡編號；componentDropsFromStage() 供前端篩選邏輯使用
