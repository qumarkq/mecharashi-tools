backpacks Collection 欄位定義
每份 Document 均自動附帶 Firestore 系統欄位（_id、createdAt、updatedAt），以下僅列業務欄位。
🔄 v1.2 更新（PLAN-009）：背包擷取腳本 新增 mainSkill 結構（SS 稀有度背包的附帶技能）、repairAmount。 assemblableArmorType 改為陣列（正向邏輯，空陣列 = 無限制），mechRestriction 已移除。
🗃️ backpacks — 背包資料
欄位型別說明
idstring唯一識別碼（API: ID）
namestring背包名稱
typeBackpackType背包種類 Enum（API: BackpackMainType）：Heal / Ammo / Interference / Invisible / BackupEquipment / MovePointAdd / Flow / Radar / EMP / Enhance / PowerAdd
rarityWeaponRarity稀有度 Enum（與武器共用）：SS / S+ / S / A / B（API: quality）
weightnumber重量，佔機甲出力（API: weight 頂層欄位）
slotWeaponEquipSlot裝配位置；背包固定為 'back'（WeaponEquipSlot.BACK），與武器共用 enum
assemblableArmorTypeAssemblableArmorType[]可裝備機甲類型陣列（正向邏輯）：[] = 無限制；["Light"] / ["Medium"] / ["Heavy"] / 複數值（API: AssemblableAirmenType，斜線分隔 → split）
repairAmountnumber修理量；非修理類背包填 0（API: AmountOfRepair）
▎mainSkill — 附帶技能（僅 SS 稀有度，API: WithPassiveSkills[0]）
 ├ idstring技能 ID（API: WithPassiveSkills[0].ID）
 ├ namestring技能名稱（API: WithPassiveSkills[0].name）
 ├ iconstring?技能圖示鍵名（API: WithPassiveSkills[0].SkillIcon / .icon）
 ├ descriptionstring技能效果描述（清洗 rich text 標籤後；API: WithPassiveSkills[0].SpecificEffects）
 ├ buffIdsstring[]觸發的 Buff ID 列表 → 引用 buffs collection（API: BufCarried，斜線分隔）
 ├ dmgnumber?增傷加成 %（管理員手動填入）
 ├ critnumber?暴擊率加成（管理員手動填入）
 ├ critDmgnumber?暴擊傷害加成 %（管理員手動填入）
 ├ accnumber?命中率加成（管理員手動填入）
 └ specialEffectsstring[]?非數值效果描述列表（管理員手動填入）
02 — Notes
設計說明
💡 assemblableArmorType 陣列設計 assemblableArmorType 為 string[]（正向邏輯），指定「哪些機甲可以裝備此背包」。 空陣列 [] = 無限制，所有機甲皆可裝備。 允許多值（如 ["Light", "Medium"]），腳本從 API AssemblableAirmenType（斜線分隔）解析。 mechRestriction（舊負向邏輯）已完全移除。
💡 mainSkill 的數值欄位 API 回傳的 SpecificEffects 為 rich text 格式，包含標籤與數值描述。 dmg / crit / critDmg / acc 等數值需管理員從描述中手動解析後填入，供傷害計算器直接使用。
