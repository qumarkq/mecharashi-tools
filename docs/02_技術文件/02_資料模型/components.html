設計說明
觸元件 / 應元件 系統
元件分為「觸元件」（觸發條件）和「應元件」（觸發後效果），插入武器的對應插槽中。
⚡ 模擬器計算範圍 遊戲中元件效果種類繁多，但模擬器僅計算以下 6 類對傷害有直接影響的效果：
① 增傷（dmgBoost）② 增加子彈（bulletAdd）③ 增加倍率（multiplierBoost）④ 破甲（armorBreak）⑤ 每消耗N點AP增傷（apDmgBoost）⑥ 對軀幹增傷（torsoDmgBoost）
其他效果（移動力、防禦相關）不在傷害模擬範圍內。
Schema — 觸元件
觸元件欄位（slot: "trigger"）
欄位型別說明
idstring唯一識別碼
namestring元件名稱（如 "觸元件-共力"）
slot"trigger"固定值，表示觸元件
levelnumber元件等級（如 Lv.4）
probabilitystring觸發概率類型："概率" / "必定"
conditionstring觸發條件描述（如 "同時使用兩把武器攻擊時"）
conditionTypeConditionType (Enum) dualWield = 同時使用兩把武器
singleWield = 使用單把武器
firstAttack = 首次攻擊
apCost = 每消耗N點AP
targetTorso = 攻擊軀幹
always = 每次攻擊
conditionValuenumber?條件參數（apCost 類型填 AP 數值）
rarityRarity (Enum)稀有度
Schema — 應元件
應元件欄位（slot: "effect"）
欄位型別說明
idstring唯一識別碼
namestring元件名稱（如 "應元件-強擊"）
slot"effect"固定值，表示應元件
levelnumber元件等級（如 Lv.5）
probabilitystring觸發概率類型："概率" / "必定"
effectTypeEffectType (Enum) dmgBoost = 增傷（傷害提升%）
bulletAdd = 增加子彈（發射子彈數+1）
multiplierBoost = 增加倍率（攻擊倍率提升）
armorBreak = 破甲（降低敵方裝甲）
apDmgBoost = 每消耗N點AP增傷
torsoDmgBoost = 對軀幹增傷
valuenumber效果數值（20 = 20%，1 = +1 子彈）
descriptionstring完整效果描述文字
rarityRarity (Enum)稀有度（金/紫/藍）
