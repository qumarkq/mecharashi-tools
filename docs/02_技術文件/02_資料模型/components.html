UPDATED v1.6 — 2026-05-24
設計說明
觸元件 / 應元件 系統
元件分為「觸元件」（Condition）和「應元件」（Function），對應 WIKI mQuery 的 ModuleType 欄位，插入武器的對應插槽中。
⚡ 模擬器計算範圍 遊戲中元件效果種類繁多，但模擬器僅計算以下 6 類對傷害有直接影響的效果：
① 增傷（dmgBoost）② 增加子彈（bulletAdd）③ 增加倍率（multiplierBoost）④ 破甲（armorBreak）⑤ 每消耗N點AP增傷（apDmgBoost）⑥ 對軀幹增傷（torsoDmgBoost）
其他效果（移動力、防禦相關）不在傷害模擬範圍內。
Schema — 觸元件
觸元件欄位（componentType: "Condition"）
欄位型別說明
idstring唯一識別碼
namestring元件名稱（如 "觸元件-共力"）
componentType"Condition"固定值，對應 WIKI ModuleType
moduleSubtypeModuleSubtype (Enum) 1=命中相關；2=攻擊方式；3=受擊相關；4=耐久相關；5=AP相關；
6=戰中效果；7=戰後效果；8=攻擊結果；9=距離相關；10=特殊效果；11=移動相關
probabilityLevelnumber元件等級（對應 WIKI ProbabilityLevel 欄位）
conditionstring觸發條件描述（如 "同時使用兩把武器攻擊時"）
conditionTypeConditionType (Enum) dualWield = 同時使用兩把武器
singleWield = 使用單把武器
firstAttack = 首次攻擊
apCost = 每消耗N點AP
targetTorso = 攻擊軀幹
always = 每次攻擊
allowedWeaponTypesWeaponType[]允許裝備的武器種類（預設全給，管理者手動刪除不符合的）
rarityItemRarity (Enum)稀有度：EX（橙）/ S（金）/ A（紫）/ B（藍）；API quality 映射：SSR→EX / SR→S / R→A / N→B
iconstring技能圖示 key（如 "Icon_skill_passive_5223"），由 API IconPath 欄位取得
iconLocalstring本機技能圖示路徑（如 "/images/components/Icon_skill_passive_5223.png"）
outerFrameLocalstring本機外框圖路徑，依 componentType × componentsWType 決定（見下方說明）；由 patch-component-outerframe.mjs 自動填入
componentsWTypeComponentsWType (Enum)W 類型標記：W = W型元件（僅限 WeaponEquipSlot 為 DUAL_HAND / BACK 的武器可裝備，傷害計算機用）；Normal = 一般元件
🖼 元件圖示雙層結構
元件圖示由兩層圖片疊合組成：
① 外框圖（outerFrameLocal 欄位）— 靜態資源，存放於 public/images/components/OuterFrame/，依 componentType × componentsWType 共四種：
 · Condition + Normal → statetype_Condition.png
 · Condition + W    → statetype_Condition_W.png
 · Function + Normal  → statetype_Function.png
 · Function + W    → statetype_Function_W.png
② 技能圖示（iconLocal 欄位，如 Icon_skill_passive_5223.png）— 由擷取腳本自動下載，存放於 public/images/components/
顯示時將技能圖示疊加於外框圖上；外框圖固定存在，技能圖示不存在時以 onError 隱藏。

本機靜態資料
Boss 掉落資料（非 Firestore）
Boss 掉落資料以靜態 TypeScript 檔案維護，不存入 Firestore。
📁 src/data/bossDrops.ts
元件 BOSS 掉落資料表，結構：BOSS_DROPS[componentType][componentsWType][shortName] → StageDrop[]
・componentType：'Function' | 'Condition'
・componentsWType：'Normal' | 'W'
・shortName：元件名稱去掉前綴，如 "應元件-強擊" → "強擊"

StageDrop 型別
{ stage: number; bosses: number[] }（關卡編號 + 可掉落的 Boss 編號陣列）

工具函式
・getBossImagePath(stage, bossNum)：回傳 Boss 頭像路徑（/images/components/Stage{N}_Boss/character_{N}_{M}.png）
・extractShortName(fullName)：從完整元件名稱擷取短名稱（去掉"應元件W?-"/"觸元件W?-"前綴）
・getComponentDrops(componentType, componentsWType, fullName)：查詢元件所有 Boss 掉落
・getAllStages()：取得資料中所有出現過的關卡編號（已排序）
・componentDropsFromStage(componentType, componentsWType, fullName, stage)：檢查元件是否有指定關卡掉落
🖼 Boss 頭像路徑規則
/images/components/Stage{N}_Boss/character_{N}_{bossNum}.png
例：第 11 關 Boss 2 → /images/components/Stage11_Boss/character_11_2.png
前端元件
BossDropTooltip / BossDropSection
元件圖鑑頁面的掉落資訊顯示元件（src/components/BossDropTooltip.tsx）。
元件使用場景說明
BossDropTooltip桌面版卡片滑鼠懸停卡片時，以 @floating-ui/react Tooltip 顯示 Boss 掉落資訊（關卡 + Boss 頭像）；行動版停用
BossDropSection行動版 BottomSheet內嵌於 BottomSheet 詳情面板底部，直接列出所有掉落來源；無掉落資料時不渲染
BossDropContent內部共用Tooltip 與 BottomSheet 共用的掉落列表渲染（關卡分組、Boss 編號 + 頭像）
