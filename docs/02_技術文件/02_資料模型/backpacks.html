Schema 更新說明
PLAN-009 背包擷取腳本（2026-05-21 確立）

· 圖示：新增 icon: string，由腳本根據 ID 自動組裝 CDN URL（v1.3）
· 稀有度：新增 rarity: WeaponRarity（與武器共用），API quality 欄位直接對應
· 背包種類強型別：type: string → 對應新建 BackpackType enum（API: BackpackMainType）
· 可裝備機甲類型：assemblableArmorType: AssemblableArmorType[] 陣列（正向邏輯，API: AssemblableAirmenType）；允許多種機甲，空陣列代表無限制；mechRestriction 已移除
· 修理量：新增 repairAmount: number（API: AmountOfRepair；非修理類背包填 0）
· 附帶技能：skill 欄位改名為 mainSkill，並補入 API 欄位（id / icon / buffIds）；僅 SS 稀有度（SSSR）背包才有此欄位

AssemblableArmorType
可裝備機甲類型（AssemblableArmorType Enum）
正向邏輯陣列：指定「哪些機甲可以裝備此背包」。空陣列 [] 代表無限制。
Enum Key值說明API 對應
LIGHTLight輕型機甲可裝備AssemblableAirmenType: "Light"
MEDIUMMedium中型機甲可裝備AssemblableAirmenType: "Medium"
HEAVYHeavy重型機甲可裝備AssemblableAirmenType: "Heavy"
欄位為陣列，允許多種機甲同時裝備
例如 ["Light", "Medium"] 代表輕型與中型皆可裝備。
[]（空陣列）= 無限制，所有機甲皆可裝備。
API AssemblableAirmenType 以斜線分隔多值（如 "Light/Medium"），腳本自動 split 後寫入陣列。
Schema
欄位定義
背包是獨立裝備，有自己的種類、稀有度、修理量，以及可裝備機甲類型限制。SS 稀有度背包另有 API 提供的附帶技能資料。
欄位型別說明
idstring唯一識別碼（API: ID）
namestring背包名稱（如 "拯救者背包"）
iconstring?背包圖示 CDN URL（由腳本根據 ID 自動組裝）
格式：https://media.zlongame.com/media/pictures/cn/community/img/gl/gameInfo/pack/Icon_backpack_{ID}.png
typeBackpackType (string)背包種類：Heal / Ammo / Interference / Invisible / BackupEquipment / MovePointAdd / Flow / Radar / EMP / Enhance / PowerAdd
API: BackpackMainType（英文值直接對應）
rarityWeaponRarity (string)稀有度（與武器共用）：SS / S+ / S / A / B
API: quality · SSSR→SS / UR→S+ / SSR→S / SR→A / R→B
weightnumber重量（佔機甲出力）
API: weight（頂層欄位；WithPassiveSkills 內的 weight 為排序用途，不可混用）
slotWeaponEquipSlot (string)裝配位置；背包固定為 'back'（WeaponEquipSlot.BACK），與武器共用 enum，供裝備計算器統一判斷部位
assemblableArmorTypeAssemblableArmorType[] (string[])可裝備機甲類型陣列（正向邏輯）：[] = 無限制；["Light"] / ["Medium"] / ["Heavy"] / 複數值
API: AssemblableAirmenType · 斜線分隔多值 → split('/') → 過濾已知型別
repairAmountnumber修理量；非修理類背包固定填 0
API: AmountOfRepair · parseInt；不存在時預設 0
▎附帶技能 mainSkill（僅 SS 稀有度，API: WithPassiveSkills[0]）
 ├ idstring技能 ID
API: WithPassiveSkills[0].ID
 ├ namestring技能名稱（如 "拯救者驅動·攻擊"）
API: WithPassiveSkills[0].name
 ├ iconstring?技能圖示鍵名（如 "Icon_skill_passive_3130"）
API: WithPassiveSkills[0].SkillIcon / .icon
 ├ descriptionstring技能效果描述（清洗 rich text 標籤後）
API: WithPassiveSkills[0].SpecificEffects · 清除 <color=#…>…</color> 後寫入
 ├ buffIdsstring[]觸發的 Buff ID 列表
API: WithPassiveSkills[0].BufCarried · '/' 分隔字串 → split('/').filter(Boolean)
 ├ dmgnumber?增傷加成 %（管理員手動填入）
 ├ critnumber?暴擊率加成（管理員手動填入）
 ├ critDmgnumber?暴擊傷害加成 %（管理員手動填入）
 ├ accnumber?命中率加成（管理員手動填入）
 └ specialEffectsstring[]?非數值效果（如 "移動方式變為漂浮"）（管理員手動填入）
Rarity
背包稀有度識別規則（共用 WeaponRarity）
背包稀有度與武器使用同一 enum，由 API quality 欄位直接對應。
Enum 值API quality說明mainSkill
SS SSSR 最高等級背包；API 提供 WithPassiveSkills 技能資料 ✓ 有（API 自動填入）
S+ UR 次高等級背包 — 無 API 資料
S SSR 標準高階背包 — 無 API 資料
A SR 標準背包（高階） — 無 API 資料
B R 標準背包（一般） — 無 API 資料
API Mapping
API 欄位對照表
擷取腳本（scripts/scrape-backpacks.js）使用的 backpack_data detail 端點欄位名稱與 Firestore / TypeScript 欄位對應。
TypeScript / Firestore 欄位 API 欄位（detail 端點） 轉換規則
id ID 直接使用字串值
name name s2t() 簡→繁轉換後寫入
icon （無直接欄位） 腳本以 ID 組裝 CDN URL：https://…/pack/Icon_backpack_{ID}.png
type BackpackMainType 英文值直接對應 BackpackType enum（Heal / Ammo / Interference / Invisible / BackupEquipment / MovePointAdd / Flow / Radar / EMP / Enhance / PowerAdd）
rarity quality SSSR→'SS' / UR→'S+' / SSR→'S' / SR→'A' / R→'B'
weight weight（頂層） parseInt(value, 10)；注意 WithPassiveSkills 內亦有 weight 欄位，為排序用途，不可混用
assemblableArmorType AssemblableAirmenType 斜線分隔多值字串 → split('/') → 過濾 Light/Medium/Heavy → 陣列；空值 → []（無限制）
repairAmount AmountOfRepair parseInt(value, 10)；欄位不存在時填 0
mainSkill.id WithPassiveSkills[0].ID 僅 quality=SSSR 時存在；陣列為空時整個 mainSkill 欄位不寫入
mainSkill.name WithPassiveSkills[0].name s2t() 簡→繁轉換後寫入
mainSkill.icon WithPassiveSkills[0].SkillIcon / .icon 鍵名格式，如 "Icon_skill_passive_3130"
mainSkill.description WithPassiveSkills[0].SpecificEffects 清洗 <color=#...>…</color> 等 rich text 標籤；s2t() 繁體化
mainSkill.buffIds WithPassiveSkills[0].BufCarried 斜線分隔字串 → split('/').filter(Boolean)
API 端點：GET https://ma-activity.zlongame.com/common/infodata/mQuery.do?appkey=1616148215678&target=backpack_data&type=detail&query={背包ID}
列表端點：type=list（取得所有背包 ID 清單後，逐一呼叫 detail）
管理員保護：腳本寫入時若 prev.mainSkill 已存在且包含手動填入的 dmg/crit/critDmg/acc，不覆蓋這些欄位。
篩選邏輯
配裝模擬器中的篩選規則
📋 背包選擇篩選邏輯
1. 機甲類型過濾：依 assemblableArmorType 判斷。若為 'Light'，選了中/重機甲的玩家看不到此背包；'All' = 無限制。
2. 重量過濾：背包 weight ≤ 機甲剩餘出力（output - 武器 weight 總和）才顯示。
3. 以上兩個條件皆由前端自動過濾，不需使用者判斷。
