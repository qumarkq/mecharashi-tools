UPDATED v2.3
v1.6 更新
重構說明
🔄 v2.3 更新：新增 moduleLevel 欄位（配裝模擬器用）  • 新增 moduleAddLevel 欄位（number，預設 1）：代表此模組在插槽中所貢獻的等級增幅
 • 配裝模擬器累加所有嵌入模組的 moduleAddLevel，以計算總模組等級
 • 遷移時對所有現有模組補寫 moduleAddLevel: 1；副模組同樣預設 1
🔄 v2.2 更新：ModuleSource 新增「機甲不可拆」、boundPart 改為陣列  • ModuleSource 新增枚舉值 '機甲不可拆'：表示模組來自機甲但該機甲無法拆解取得（source 仍可與其他值並存於陣列中）
 • boundPart 由單一字串改為 MechPartPosition[] 陣列，支援模組同時綁定多個部位（null 或空陣列 = 不限部位）
 • 管理後台 source 選取任何「非未知」來源時，自動取消「未知」；boundPart 改為複選核取方塊
🔄 v2.1 更新：source 改為陣列、新增 dismantleMechIds 欄位  • source 由單一字串改為 ModuleSource[] 陣列，支援複數取得途徑（如同時可在商店購買也可拆機甲）
 • 新增 dismantleMechIds 欄位（string[]?）：拆解哪些機甲可取得此模組，儲存機甲 ID 陣列（對應 mechs collection），顯示時 mapping 為機甲名稱
 • 管理後台 source 欄位改為複選核取方塊；dismantleMechIds 以機甲名稱清單複選
🔄 v2.0 更新：新增武器專屬增傷欄位（20 個）  • 新增 4 個武器類別增傷欄位：dmg_assault（突擊）/ dmg_melee（格鬥）/ dmg_shooting（射擊）/ dmg_tactical（戰術）
 • 新增 15 個武器種類增傷欄位：dmg_blade（刀劍）/ dmg_polearm（長柄）/ dmg_missile（導彈）/ dmg_rocket（火箭）/ dmg_shotgun（霰彈）/ dmg_machinegun（機槍）/ dmg_heavy_machinegun（重機槍）/ dmg_railgun（電磁炮）/ dmg_funnel（浮游炮）/ dmg_sniper_light（輕型狙擊步槍）/ dmg_sniper（狙擊步槍）/ dmg_fist（拳套）/ dmg_pile（打樁機）/ dmg_chainsaw（電鋸）/ dmg_flamethrower（噴火器）
 • 新增 2 個特殊情境增傷欄位：dmg_counter（反擊）/ dmg_enemy_phase（敵方階段）
 • 所有新欄位為選填（number?），預設 0；同樣適用於 levels[] 各等級資料；conditionalEffects[].stats 亦可引用這些欄位鍵名
🔄 v1.9 更新：新增 conditionalEffects[] + ConditionalTrigger enum（傷害模擬器用）  • 新增 conditionalEffects 欄位（ConditionalEffect[]），儲存條件性效果供傷害模擬器運算
 • 新增 ConditionalTrigger 列舉：perBuffHeld / perCombo / apSkill / perAmmoConsumed / tacticalWeapon
 • 無條件效果的模組此欄位為空陣列 []；副模組同樣為 []
🔄 v1.8 更新：全欄位 enum 化 + source/managedBy 語義釐清 + slot 資料遷移  • slot 全面改用 ModuleSlot 列舉值（'機甲特性模組'/'機甲8級模組'/'通用模組'/'機甲副模組'）；modules.json 已完成資料遷移
 • source 語義回歸遊戲取得途徑（ModuleSource：商店/拆機甲/未知）
 • 新增 managedBy 欄位（ModuleDataSource：auto/manual）專門表示後台資料維護方式，與 source 分離
🔄 v1.7 更新：ModuleRarity 列舉 + 管理後台 enum 防呆  • rarity 欄位改用獨立的 ModuleRarity 列舉（S / A），不再沿用通用的 ItemRarity
 • boundPart 欄位在後台表單改由 MechPartPosition 列舉驅動，避免打錯字串
🔄 v1.6 重構：雙來源擷取 + slot 重新定義 + 各等級效果 slot 五種類型：
 • "機甲特性模組" — 需 4 模組槽；來自 module_data API；max 4 levels；名稱不帶 Ⅰ/Ⅱ
 • "機甲8級模組" — 需 8 模組槽；來自 module_data API；max 8 levels
 • "通用模組" — 來自 module_data API；max 4 levels；名稱含 Ⅰ/Ⅱ；可通用裝備
 • "機甲副模組" — 機甲自帶；來自 aircraft_data API（含 S+A 品質機甲）；模組圖鑑預設不顯示（可勾選「顯示副模組」開啟）
 • "機甲專屬模組" — 機甲限定；永遠排除於模組圖鑑，僅在機甲詳情頁與配裝模擬器中呈現

模組 ID 格式：
 • 特性/8級/通用模組：mod_{masterApiId}（如 "mod_1001"）
 • 副模組：mod_{機甲名}_{position}（如 "mod_都卜勒_4mod"）

▎條件效果 conditionalEffects[]（傷害模擬器用；無條件效果或副模組為空陣列）
 ├ triggerConditionalTrigger (Enum)觸發類型：perBuffHeld（每個增益）/ perCombo（每連擊）/ apSkill（AP技能）/ perAmmoConsumed（每消耗彈藥）/ tacticalWeapon（使用戰術武器）
 ├ minCountnumber?觸發門檻值（apSkill 時代表最低 AP 數，如 3）
 ├ statsstring[]受影響的屬性欄位名稱陣列（如 ["dmg"]、["dmg","acc_rate"]）；直接對應 module 頂層欄位名
 ├ basenumber?觸發條件成立時的基礎加成值（未設定則為 0）
 ├ scalenumber?每單位追加值（每個 buff / 每 1 連擊 / 每超額 1 AP / 每 1 彈藥）
 ├ maxnumber?總加成上限（base + scaling 合計不超過此值）
 ├ maxStacksnumber?最多疊加層數（perAmmoConsumed 等疊層效果用）
 ├ durationnumber?效果持續回合數；null / 未設定 = 永久（直到戰鬥結束）
 └ resetOn"attack" | "turn" | null效果重置時機："attack"（每次攻擊後重置，如連擊數）/ "turn"（回合結束）/ null（不重置）
