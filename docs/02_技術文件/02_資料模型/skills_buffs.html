NEW v2.0
設計說明
skills Collection
可跨實體引用的共用技能定義庫。
🆕 v2.0 新增：skills Collection 機師的 talents / skills / neuralDrive 仍以嵌入方式存於 pilots 文件中（讀取效率優先）。
skills Collection 存放可被多個實體引用的技能模板（如武器/模組共用技能效果）。
欄位含 tags[]（分類標籤）、buffIds[]（技能觸發的 BUFF 列表）等。
欄位型別說明
idstring唯一識別碼（如 "skill_001_增傷"）
namestring技能名稱
typestring技能類型（被動技能/主動技能）
activationSkillActivation (Enum)?生效方式（武器技能適用）：carry / equip / use
descriptionstring效果描述文字
dmg / crit / critDmg / accnumber可量化的加成數值
tags[]string[]分類標籤（如 ["增傷", "被動"]）
buffIds[]string[]此技能觸發的 BUFF ID 列表（關聯 buffs Collection）
Schema
buffs Collection
🆕 v2.0 新增：buffs Collection（初始版本） BUFF 定義庫，任何技能、模組、武器均可透過 buffIds[] 陣列引用。
初始版本僅含基本描述欄位，後續版本再擴充效果計算欄位。
欄位型別狀態說明
idstring✅ v1.0唯一識別碼（如 "buff_001_增傷"）
namestring✅ v1.0BUFF 名稱（如 "護甲強化"）
descriptionstring✅ v1.0BUFF 效果描述文字
iconstring?✅ v1.0圖示路徑（可留空）
durationnumber?🔜 預留持續回合數，null = 永久
maxStacknumber?🔜 預留最大疊加層數
effects[]array?🔜 預留具體效果列表 — 待設計
buffTypestring?🔜 預留BUFF 類型分類（增益/減益/控制/護盾）— 待枚舉
⚠️ 後續擴充 v1.0 先建立基本架構（id + name + description + icon），確保技能可以引用 BUFF ID。
待遊戲 BUFF 機制研究完整後，再補充 duration / maxStack / effects[] / buffType 等計算欄位。
