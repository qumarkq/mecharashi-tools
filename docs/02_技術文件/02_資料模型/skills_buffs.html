共用 Interface（SkillCondition · SkillEffect）· 可跨實體引用的技能模板 · BUFF 定義庫

PLAN-001 v3.0
Shared Interface
共用 Interface
SkillCondition 與 SkillEffect 為跨實體共用的基礎型別，定義於 src/types/index.ts。
🆕 PLAN-001：新增 SkillCondition、SkillEffect、GameBuff interface 取代 PilotSkill / PilotTalent / NeuralDriveLevel / WeaponSkill 原有的平坦數值欄位（dmg / crit / critDmg / acc）。
condition 欄位僅用於顯示說明標籤，計算器不自動判斷觸發。
SkillCondition
技能效果的觸發條件說明，顯示給玩家作為標籤，不驅動計算器自動邏輯。
欄位型別說明
triggerPilotSkillTrigger (Enum)觸發類型（顯示說明用，計算器不自動判斷）：
always 永久被動 / onAttack 攻擊時 / onCounter 反擊時 / onApSkill 使用 AP 技能時
weaponCategory 使用特定武器類別 / dualWield 雙持 / hpBelow HP 低於門檻
firstAttack 戰鬥首次攻擊 / enemyPhase 敵方回合 / allyHasBuff 隊友持有增益
weaponCategorystring?限定武器類別（WeaponCategory enum）；trigger = weaponCategory 時填入
hpThresholdnumber?HP% 門檻（如 50 代表 "HP < 50%"）；trigger = hpBelow 時填入
minApCostnumber?最低 AP 消耗條件
targetClassstring?目標職業限制（PilotClass enum）
SkillEffect
單一可計算效果條目，stat 值與 Module 平坦欄位名稱對齊以確保計算器加總邏輯一致。
欄位型別說明
statstring影響屬性，與 Module 欄位名稱對齊：
'dmg' | 'crit' | 'critDmg' | 'acc'
'dmg_assault' | 'dmg_melee' | 'dmg_shooting' | 'dmg_tactical'
'dmg_blade' | 'dmg_machinegun' | ...（武器種類）
'range' | 'armor_rate' | 'firepower_rate' | ...（其他屬性）
valuenumber數值大小（如 15 代表 15%）
scopeSkillScope (Enum)受益對象：self（自己）/ ally（其他隊友）/ team（全隊含自己）
conditionSkillCondition | null觸發條件說明；null = 此技能 active 時永遠生效
空陣列語義 effects: [] 代表「技能存在，但數值待管理員填入」，計算器遇到空陣列直接跳過，不影響運作。
不再使用 dmg: 0, crit: 0, critDmg: 0, acc: 0 這種有誤導性的初始值。
SkillType Enum — PilotSkill.type 對應值 被動技能 — 永久生效（passive）
主動技能 — 主動使用，消耗 AP（含武器技能 EquipmentSkill）
指令技能 — 有 CD（冷卻回合），額外欄位 cd?: string，對應 API type = "Order"
設計說明
skills Collection
可跨實體引用的共用技能定義庫。機師技能仍嵌入 pilots 文件（讀取效率優先）。
🆕 v2.0 / PLAN-001：skills Collection 機師的 talents / skills / neuralDrive 仍以嵌入方式存於 pilots 文件中（讀取效率優先）。
skills Collection 存放可被多個實體引用的技能模板（如武器 / 模組共用技能效果）。
PLAN-001 起以 effects: SkillEffect[] 取代舊有的平坦數值欄位。
欄位型別說明
idstring唯一識別碼（如 "skill_001_增傷"）
namestring技能名稱
typestring技能類型（被動技能 / 主動技能）
activationSkillActivation (Enum)?生效方式（武器技能適用）：carry / equip / use
descriptionstring效果描述文字
effectsSkillEffect[]可量化效果列表（取代舊平坦數值欄位）
tags[]string[]分類標籤（如 ["增傷", "被動"]）
buffIds[]string[]此技能觸發的 Buff ID 列表（關聯 buffs Collection）
Schema
buffs Collection — GameBuff
🆕 PLAN-001：GameBuff interface 完整定義 BUFF 定義庫，任何技能、模組、武器均可透過 buffIds[] 陣列引用。
玩家可在計算器中獨立切換 Buff（不依賴技能開關），計算器加總所有 active Buff 的 effects[]。
欄位型別狀態說明
idstring✅ v1.0唯一識別碼（如 "buff_過熱狀態"）
namestring✅ v1.0Buff 名稱（如 "過熱狀態"）
descriptionstring✅ v1.0Buff 效果描述文字
iconstring?✅ v1.0圖示路徑（可留空）
buffTypeBuffType (Enum)✅ PLAN-001Buff 類型：statBoost / resource / state / debuff / control
maxStacknumber?✅ PLAN-001最大疊加層數；resource 類型用，省略表示不可疊加
durationnumber?✅ PLAN-001持續回合數；null = 持續到消耗 / 手動關閉
effectsSkillEffect[]✅ PLAN-001Buff active 時提供的效果列表
BuffType 枚舉值說明 statBoost — 純數值增益（最常見）
resource — 可堆疊資源（如彈潮蓄能、連擊層數），搭配 maxStack 使用
state — 特殊狀態（如過熱、強化模式）
debuff — 對敵減益
control — 控制效果
