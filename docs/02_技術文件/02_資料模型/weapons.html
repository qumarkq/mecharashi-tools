Schema 重構說明
PLAN-003 武器資料模型優化（2026-05-18 實施）

· 射程結構化：移除 range: string，改為三欄位：rangeType + minRange + maxRange
· 術語對齊：category → type（類型），type → kind（種類），typeCoefficient → kindCoefficient
· 稀有度強型別：rarity: string → rarity: WeaponRarity（SS / S+ / S）
· 機甲限制：新增 mechRestriction: MechRestriction 欄位
· SkillCondition：weaponCategory? → weaponType?，新增 weaponKind?
Terminology
類型 vs 種類 術語說明
武器分類有兩個層級，欄位命名與中文概念對齊。
type（類型）
大分類 — WeaponType enum
射擊 / 格鬥 / 突擊 / 戰術
kind（種類）
細分類 — WeaponKind enum
格鬥類 7 種 / 戰術類 4 種 / 突擊類 4 種 / 射擊類 2 種
種類是模組條件篩選的關鍵維度
模組 SkillCondition 使用 weaponKind? 限定到種類層級（如「機槍」、「狙擊步槍」）， 讓計算器能正確判斷模組是否對當前武器生效。 使用 weaponType? 則限定到大分類（如「射擊」、「格鬥」）。

Range
射程型態說明
武器射程有三種型態，對應不同的遊戲機制。
manhattan — 菱形射程
採 Manhattan 距離（|dx|+|dy|），可打斜格，但斜格計距更遠。
攻擊格：minRange ≤ |dx|+|dy| ≤ maxRange
大多數武器使用此型態（機槍、狙擊、霰彈、格鬥等）。
例："2-4" → rangeType:'manhattan', minRange:2, maxRange:4
例："1-3" → rangeType:'manhattan', minRange:1, maxRange:3
orthogonal — 十字直線
只能攻擊上下左右四個方向的直線格，不可打斜格。
攻擊格：(dx=0 XOR dy=0) AND minRange ≤ |dx|+|dy| ≤ maxRange
目前僅電磁炮使用。特定角色技能可解鎖斜格攻擊（射程 -1）。
例：電磁炮 → rangeType:'orthogonal', minRange:0, maxRange:5
ring — 環形 N 圈
以持有者為中心，向外擴展 N 圈的方形覆蓋區域。
N 圈 = (2N+1)×(2N+1) 方格（含自身格，無最小限制）。
採 Chebyshev 距離（水平、垂直、斜角均計 1 格）。
例："2+" → rangeType:'ring', minRange:0, maxRange:2
例："1+" → rangeType:'ring', minRange:0, maxRange:1

0 = 持有者位置；■ = 攻擊範圍格；2圈 = 5×5 = 25 格（含自身）
計算機開發備注：
· ring 判斷：Math.max(Math.abs(dx), Math.abs(dy)) <= maxRange
· manhattan 判斷：minRange <= Math.abs(dx) + Math.abs(dy) && Math.abs(dx) + Math.abs(dy) <= maxRange
· orthogonal 判斷：(dx === 0 || dy === 0) && dx !== dy && minRange <= Math.abs(dx) + Math.abs(dy) && Math.abs(dx) + Math.abs(dy) <= maxRange

SkillCondition / SkillEffect
條件與效果型別擴充（PLAN-005）
為支援專武天賦強化的結構化計算，PLAN-005 擴充了共用型別。
SkillCondition — 新增欄位
欄位型別說明
hasBuffstring?trigger = 'hasBuff' 時填入：需持有的 buff / 狀態名稱（如 '強化射擊'、'瞄準'）
SkillEffect — 新增欄位
valueType'add' | 'override'計算方式：'add'（預設，加算）/ 'override'（覆蓋原始值）
用於「技能倍率 0.15 → 0.2」這類覆蓋現有值而非加算的效果
trigger 值中文語義對應情境額外欄位
always無條件持有 / 裝備即生效—
onAttack攻擊時主動攻擊觸發—
onCrit造成暴擊後專武技能「熱力連珠」—
hasBuff持有特定狀態需持有 buff / 狀態（如「強化射擊」）hasBuff: string
notMoved本回合未移動專武技能「白鶴之眺」暴擊率條件—
hpBelowHP 低於門檻HP 條件效果hpThreshold: number

Rarity
武器稀有度識別規則
WeaponRarity enum 共五級，由 API quality 欄位直接對應，與通用 ItemRarity 無關。
Enum 值API quality識別規則說明
SS SSSR API quality = SSSR，isExclusive = true 專屬武器。遊戲中最高等級武器。
S+ UR API quality = UR；名稱通常含「改」字 改版武器，一般為通用高階武器的強化版本。
S SSR API quality = SSR；名稱通常含「EX」字樣 EX 版本武器，標準高階武器。
A SR API quality = SR 標準武器（高階）。
B R API quality = R 標準武器（一般）。
對應邏輯：爬蟲腳本以 quality 欄位為主要來源（RARITY_FROM_API）； 若 quality 欄位缺失，fallback 使用名稱規則（含「改」→ S+、含「EX」→ S）。
Guide
專武被動效果填值指引
專屬武器（isExclusive = true）的條件式被動效果，以 skills[] 中 activation='carry' 的技能條目表達。
填值原則
不新增額外欄位。專武條件式被動效果 = skills[] 中一筆 activation: 'carry' 的 WeaponSkill。 其 effects: SkillEffect[] 依模組效果格式填入結構化數值。
計算器處理武器 carry 技能的邏輯與模組效果完全相同，不需要額外的處理分支。
// 專武「昏星繼光」的被動效果：對輕型機甲攻擊時傷害 +15% // → 以 activation='carry' 的 WeaponSkill 條目表達 { name: "昏星繼光‧被動", type: '被動技能', activation: 'carry', description: "攻擊輕型機甲時，傷害提升 15%", effects: [ { stat: 'dmg', value: 15, scope: 'self', condition: { trigger: 'always', targetClass: 'light' // 目標為輕型機甲 } } ], buffIds: [] }
管理員作業優先序
1. 確認所有 isExclusive = true 的武器，skills[] 中已有 activation:'carry' 的條目
2. 目前僅有文字描述（description）而缺 effects[] 結構化數值的條目優先處理
3. 長期持續補填，不阻塞功能開發
