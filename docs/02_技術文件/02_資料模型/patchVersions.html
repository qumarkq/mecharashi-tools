patchVersions版本時間線資料模型

靜態 TypeScript 資料檔，定義版本資訊（PatchVersion）、半版本（PatchHalf）、計時活動（TimedActivity）的完整欄位規格。PLAN-014 Phase A 擴充。

Overview
資料來源 & 用途
位置：src/data/patchVersions/

組成：每個版本各一份 v{N}.ts；型別定義集中於 types.ts； index.ts 匯總 PATCH_VERSIONS 陣列供前台元件使用。

用途：首頁版本時間線（VersionTimeline）、版本濃縮資訊表（VersionQuickTable）、 展開面板（VersionExpandedPanel）、甘特圖活動時間線（ActivityTimeline）。

PLAN-014 Phase B 計畫將此靜態資料遷移至 Firebase Firestore，並以 usePatchVersions() hook 取代靜態 import，屆時本文件中的欄位定義亦適用於 Firestore Document 結構。
PatchVersion
PatchVersion — 版本頂層
export interface PatchVersion { version: string // 如 "2.8"、"3.3" name?: string // 版本副標，如 "黃昏紀元" bannerImage?: string // 版本前瞻圖路徑（/images/banners/v2.8.jpg） upper: PatchHalf // 上半版本 lower: PatchHalf // 下半版本 crisisShop?: string[] // 危境重構可兌換機師 memoryStorm?: string // 記憶風暴說明 borderShop?: string // 邊境商店更新（如 "雷克斯(6/15)"） arenaShop?: string // 鬥技場更新（如 "銀閃(6/18)"） grayOpsUpdates?: GrayOpsUpdate[] // 灰燼行動機甲列表更新 notes?: string // 版本備註 isTwCurrent?: boolean // 此版本是否為台服當前版本（首頁標記用） }
欄位型別必填說明
version string 必填 版本號字串，如 "2.8"、"3.3"。PLAN-014 Phase B 將作為 Firestore Document ID。
name string? 選填 版本副標題，如 "黃昏紀元"，顯示於版本徽章與頁頭。
bannerImage string? 選填 版本前瞻圖路徑，相對於 public/，如 /images/banners/v2.8.jpg。
isTwCurrent boolean? 選填 台服當前版本標記，全局只有一個版本為 true，於首頁顯示 ★ 標記。
crisisShop string[]? 選填 危境重構商店中此版本新增的機師名稱列表。
grayOpsUpdates GrayOpsUpdate[]? 選填 灰燼行動公司機甲列表更新，參見 GrayOpsUpdate 型別。
PatchHalf
PatchHalf — 半版本（上半 / 下半）
export interface PatchHalf { cnDate: string // 陸服起始日 YYYY/MM/DD（固定週四） twDate?: string // 台服起始日（可含 "約 " 前綴） twIsPredicted?: boolean // 台服日期是否為預測值 pilots?: string[] // 本半版本新機師 mechs?: string[] // 本半版本新機甲 pilotSelection?: string[] // 角雕特遣機師列表（舊欄位，改用 cnActivities） mechSelection?: string[] // 跨域海運機甲列表（舊欄位，改用 cnActivities） armamentRaids?: ArmamentRaid[] // 武裝討伐副本 battlePass?: BattlePass // 版本戰令 // ── PLAN-014 Phase A 新增：甘特圖活動 ── cnActivities?: TimedActivity[] // 陸服活動（起始日 + 週數） twActivities?: TimedActivity[] // 台服活動（獨立維護，與陸服無關） // ── 舊欄位（deprecated）：cnActivities/twActivities 存在時優先使用 ── /** @deprecated */ skinGacha?: string /** @deprecated */ rouletteEvent?: boolean /** @deprecated */ specialEvents?: string[] /** @deprecated */ revivedBanners?: string[] }
陸台分離原則

同一活動在陸服與台服的時間不同，且台服有時調整順序，因此 cnActivities 與 twActivities 各自獨立填寫，不從陸服日期自動推算台服。

遷移策略：舊欄位（skinGacha、rouletteEvent 等）保留以維持向下相容； UI 優先讀取 cnActivities / twActivities，若不存在則 fallback 顯示舊欄位的文字清單。 Phase B 完成後可逐步移除舊欄位。
TimedActivity
TimedActivity — 計時活動（甘特圖單筆）
設計背景：鋼嵐所有活動均以星期四為起訖單位。 活動可用「起始日（週四）+ 持續週數」完整描述，結束日公式為 endDate = startDate + weeks × 7 − 1 天（即下週三）。 UI 元件自動計算結束日，不需手動填入。
export interface TimedActivity { name: string // 活動名稱，如「角雕刮刮樂」、「環島物流節」 startDate: string // 起始日 YYYY/MM/DD，必須為星期四 weeks: number // 持續週數（1 = 當週四到下週三） type: ActivityType // 決定甘特條顏色 pilots?: string[] // pilotMission 型別時的機師列表 mechs?: string[] // crossShipping 型別時的機甲列表 }
欄位型別必填說明
name string 必填 活動名稱，顯示於甘特條 hover tooltip，如「角雕刮刮樂」、「白夜凍鋒（復刻）」。
startDate string 必填 活動起始日，格式 YYYY/MM/DD，必須為星期四（遊戲固定週四更新）。
weeks number 必填 持續週數，最小值 1。UI 自動計算 endDate = startDate + weeks × 7 − 1。
type ActivityType 必填 活動類型，決定甘特條顏色，詳見下方 ActivityType 一覽表。
pilots string[]? 選填 僅 pilotMission 類型時填入，顯示於 tooltip，如 ['白夜凍鋒', '十字線上的明光']。
mechs string[]? 選填 僅 crossShipping 類型時填入，顯示於 tooltip。
ActivityType
ActivityType — 活動類型與甘特條顏色
export type ActivityType = | 'skinGacha' // 角雕刮刮樂 | 'roulette' // 角雕輪盤 | 'pilotMission' // 角雕特遣（含機師列表） | 'crossShipping' // 跨域海運（含機甲列表） | 'specificPilotBanner' // 特定角色限定卡池 | 'specificMechBanner' // 特定機甲限定卡池 | 'limitedEvent' // 限時活動（環島物流節等） | 'loginEvent' // 限時簽到活動（碼區補給等） | 'battlePass' // 版本戰令
ActivityType顏色甘特條簡稱說明
skinGacha accent-orange 刮 角雕刮刮樂，通常持續 2 週
roulette accent-yellow 輪 角雕輪盤，通常在版本末週
pilotMission accent-purple 特 角雕特遣，需填 pilots 列表
crossShipping #c9a0dc 海 跨域海運，需填 mechs 列表
specificPilotBanner #79c0ff 角池 特定角色限定卡池
specificMechBanner #58a6d4 甲池 特定機甲限定卡池
limitedEvent accent-cyan 限 限時活動（環島物流節、回盤戰線等邏輯類活動）
loginEvent accent-green 簽 限時簽到活動（碼區補給等）
battlePass gray-500 令 版本戰令（通常貫穿整個半版本）

Gantt Rules
甘特圖週列計算規則

ActivityTimeline 元件計算邏輯

  • 週單位：以週四作為每週起始日（weekStart），一週 = 週四到下週三
  • 活動結束日：endDate = startDate + weeks × 7（exclusive）
  • 週格是否啟用：actStart ≤ weekStart < actEnd
  • 上半週列範圍：從 upper.cnDate（或 twDate）到 lower.cnDate（若活動延伸則擴展）
  • 下半週列範圍：從 lower.cnDate 延伸到最後一個活動的結束日（最少 3 週）
  • 跨半版本活動：上半活動延伸到下半的週格會在分界線兩側連續顯示色塊，不需特殊處理
  • Tracks（欄道）：upper + lower 所有 activities 合併為欄道列表；上/下半共用同一組欄道
台服 twDate 含 "約 " 前綴

台服預測日期格式為 "約 2026/06/25"，解析時需 strip 前綴後再拆解 YYYY/MM/DD。parseDate() 已處理此情況：str.replace(/^[^0-9]+/, '')。
