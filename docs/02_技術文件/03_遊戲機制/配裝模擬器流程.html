Step 1–8 操作流程、自動篩選邏輯、匯出分享圖規格

v1.5 — 配裝與傷害計算分離
分期說明
實作分期計畫
🔄 v1.5 調整：配裝與傷害計算分離 Phase 3 先實作配裝選擇 UI + 匯出分享圖模板（含元件/模組/機甲部位/武器展示），不含傷害計算。
傷害計算（公式引擎、A/B 對比、武器改裝效果計算、科研加成計算）安排於最終 Phase 8 實作。
操作流程
Step 1–8 選配流程
STEP 1
選擇機師
自動帶入六維、技能、天賦、神經驅動、駕駛等級。駕駛等級決定可選機甲範圍。
STEP 2
選擇機甲
根據機師駕駛許可篩選可選機甲。自動帶入火力、副模組、裝甲類型、出力。
STEP 3
選擇武器
根據機甲出力篩選（重量限制）。顯示所有技能與生效方式。專武天賦加強自動標記。
STEP 4
選擇背包
根據機甲裝甲類型篩選（如「僅限輕型」）。根據剩餘出力（出力 - 武器重量）篩選重量。
STEP 5
機師個別科研
機師/機甲/武裝特質各 I/II/III 欄位，有互斥提示。影響機師加成與傷害計算。
STEP 6
武器改裝
固定改裝自動帶入。浮動改裝由使用者選擇已解鎖的效果（含條件觸發效果）。
STEP 7
配置元件（觸/應）
根據武器的觸元件/應元件插槽數量，選擇觸發條件與效果。模擬器僅計算 6 種效果類型。
STEP 8
敵方 + 戰鬥參數
敵方機師/機甲、技能倍率、連擊數、是否最佳射程、AP 消耗量。（Phase 8 實作）
RESULT
即時計算結果
增傷% / 爆率% / 爆傷% / 命中% / 期望值，支援 A/B 對比與匯出圖片。（Phase 8 實作）
自動篩選
篩選邏輯
⚡ 自動篩選邏輯 選了機甲後，武器和背包會自動根據「重量 ≤ 剩餘出力」和「裝甲類型限制」過濾不可裝備的選項。
使用者不需要自己判斷能不能裝，介面直接過濾。
篩選點條件篩選依據
機甲選擇機師駕駛許可pilots.license ≥ mechs.armorType 等級
武器選擇機甲出力限制weapons.weight ≤ mechs.output
背包選擇機甲裝甲類型限制backpacks.mechRestriction == null || == mechs.armorType
背包選擇剩餘出力限制backpacks.weight ≤ mechs.output - weapons.weight
匯出功能
匯出分享圖模板（Phase 3 優先）
📸 分享圖規格 尺寸：1200×675px，適合社群分享（16:9）。
風格：仿遊戲深色卡片，橘色/藍色強調色。
包含：機師立繪 + 機甲名 + 武器 + 背包 + 模組配置 + 元件 + 網站 Logo。
技術：html2canvas 截圖 → PNG。手機端原生分享，PC 端下載。
傷害數據欄位預留空位，待 Phase 8 傷害計算完成後填入。
