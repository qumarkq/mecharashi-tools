方針確認：CSS Only，不走獨立 mobile 頁面路線
獨立 mobile 頁面需要 user-agent 偵測或 useMediaQuery 雙路由， 維護成本高、平板等中間尺寸容易誤判，且兩份頁面的資料邏輯（hooks、filters）需同步維護。

本計畫採用 Tailwind CSS 響應式前綴（sm: / md: / lg:） 統一解決， 一份元件同時服務手機、平板、桌面，無需 JavaScript 判斷裝置類型。

Design
設計決策
決策一：PilotDetailPage Hero 區塊改為手機直排、桌面三欄
手機（預設）：肖像 → 基本資訊 → 雷達圖由上而下直排。
桌面（lg:）：回到原始三欄 grid-cols-[160px_1fr_auto] 並排布局。

雷達圖在手機上改為 w-full max-w-xs mx-auto，讓它置中縮放而非固定 256px。
決策二：天賦 tab「列表 + 專武面板」改為手機直排、桌面並排
手機：天賦列表在上，專武面板在下（flex-col）。
桌面（lg:）：回到原始並排（lg:flex-row），專武面板 lg:w-72 lg:flex-shrink-0。

手機下的專武面板寬度改為 w-full，不再固定 288px。
決策三：PilotsPage 卡片底部資訊區，手機下隱藏專武名稱文字
2 欄卡片已經很窄，專武名稱 line-clamp-2 效果差。
手機下只保留 WeaponIcon（圖示），文字標籤改為 hidden sm:block，hover tooltip 已足夠。
桌面 3 欄以上才顯示名稱文字（圖示 + 文字並存的現有設計）。
決策四：AP 三卡允許換行
現有 flex gap-4 改為 flex flex-wrap gap-3， 讓三張 AP 卡在手機上能以 2+1 或 3 欄自然排列，不截斷文字。
決策五：不新增元件，只改 className
本計畫屬於純排版調整，不引入新 hook、新元件或新抽象。 所有改動集中在 PilotsPage.tsx 和 PilotDetailPage.tsx 的 Tailwind className 字串， 方便 code review 和 revert。

Breakpoints
斷點策略說明
只用 lg:（1024px）作為手機 / 桌面的分界，不用 sm: / md:

· 375px～767px（手機）：直排布局
· 768px～1023px（平板直式）：與手機相同——平板橫向空間仍不足以並排 160px 肖像 + 雷達圖
· 1024px+（平板橫式 / 桌面）：回到現有三欄並排

平板在直式使用時維持直排，在橫式使用時享有桌面版布局，符合多數使用情境。
此策略不會改動 PilotsPage 現有的 md:grid-cols-3 lg:grid-cols-4 卡片欄數設定。
