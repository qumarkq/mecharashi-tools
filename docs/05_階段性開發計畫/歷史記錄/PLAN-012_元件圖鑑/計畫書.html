Requirements
需求定義
核心需求
  • 列表頁 /components：觸元件 / 應元件 / 全部 三模式切換
  • 篩選：稀有度（EX/S/A/B）、conditionType（觸元件）/ effectType（應元件）、W 類型、關鍵字搜尋
  • 元件卡片：雙層圖示（框架圖 + 技能圖示疊合）、名稱、稀有度標籤、類型標籤、觸發條件 / 效果類型標籤
  • 描述文字中數字與 % 以醒目色標示（複用 highlightNumbers()）
  • 行動裝置：點擊卡片開啟 BottomSheet 詳細資訊面板
  • Header 導覽列新增「元件」入口（與「模組」同區塊）
⚠️ 元件圖示雙層渲染（已在 AdminPage.tsx 驗證可用）
元件圖示由兩層 <img> 以 position: relative/absolute 疊合，容器固定尺寸（圖鑑頁建議 48×48px 或依格線縮放）。

① 外框圖（底層）
路徑優先取 comp.outerFrameLocal（由 patch-component-outerframe.mjs 腳本在資料遷移時自動寫入）；若欄位為空則以下列公式推算：
/images/components/OuterFrame/statetype_{componentType}{_W}.png
外框依 componentType × componentsWType 共四種：
  • statetype_Condition.png — 觸元件 · 一般（componentsWType = "Normal"）
  • statetype_Condition_W.png — 觸元件 · W型（componentsWType = "W"）
  • statetype_Function.png — 應元件 · 一般
  • statetype_Function_W.png — 應元件 · W型
CSS：absolute，width: 80%; height: 80%，transform: translate(-50%, -50%)（置中）。外框圖固定存在，不需 fallback。

② 技能圖示（上層）
取 comp.iconLocal（完整本地路徑，如 /images/components/Icon_skill_passive_5223.png，由擷取腳本下載後填入）。
注意：icon 欄位僅為 API key 字串，不可直接當作圖片路徑使用，必須讀 iconLocal。
CSS：absolute，width: 48%; height: 48%，transform: translate(-51%, -52%) rotate(16deg)（略微偏移並旋轉以貼合框架外形設計）。
技能圖示不存在時 onError → display:none 隱藏，避免破圖。
Architecture
頁面架構設計

篩選區域

  • 類型切換標籤（觸元件 / 應元件 / 全部）— 主分類，固定於頁頂
  • 稀有度篩選（EX / S / A / B）
  • conditionType 篩選（觸元件模式用）
  • effectType 篩選（應元件模式用）
  • W 類型篩選（W 型 / 一般）
  • 關鍵字搜尋（名稱 / 描述）

元件卡片

  • 雙層圖示（CSS position: relative/absolute 疊合）
  • 名稱 + 稀有度色框
  • componentType 標籤（觸元件 / 應元件）
  • W 型標籤（若 componentsWType = "W"）
  • conditionType 或 effectType 標籤
  • condition / description 效果文字（高亮數字）

行動裝置 BottomSheet

  • 點擊卡片觸發
  • 完整顯示：圖示、名稱、所有標籤
  • 完整描述文字、武器限制（allowedWeaponTypes）
  • probabilityLevel（元件等級）
  • moduleSubtype 分類標籤

桌面版

  • 格線卡片：1 欄（手機）/ 2 欄（平板）/ 3 欄（寬螢幕）
  • 卡片直接展示完整描述
  • 篩選區固定於內容區頂部，不含側欄
  • 與模組圖鑑相同佈局模式
