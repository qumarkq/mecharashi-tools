Problem
問題背景
背包資料目前完全依賴管理員人工填入，缺乏自動化擷取流程。
問題一：背包資料無自動化擷取腳本
武器（scrape-weapons.js）、機甲（scrape-mechs.js）、模組（scrape-modules.js）、機師（scrape-pilots-v3.js） 均已有對應擷取腳本，唯獨背包需完全人工維護。 隨著遊戲更新背包數量增加，手工維護成本高且容易遺漏。
問題二：Schema 欄位未對齊 API 回傳
舊 Backpack interface 的 type: string（效果分類標籤）與 API 的 BackpackMainType 並未明確對應； mechRestriction 為負向邏輯，而 API 回傳的 AssemblableAirmenType 是正向邏輯（可裝備的機甲類型）， 兩者語義相反，需建立新欄位。 此外缺少 rarity（稀有度）和 repairAmount（修理量）欄位。
問題三：SS 稀有度背包的技能資料無法自動取得
API 對 SSSR 品質（SS 稀有度）的背包會返回 WithPassiveSkills[] 陣列， 包含技能 ID、名稱、效果描述、Buff ID、圖示等結構化資料。 目前這些資料需管理員人工逐一填入，且舊的 skill 欄位結構未能映射 API 欄位。
Design
設計決策
決策一：新增 BackpackType enum
建立 BackpackType enum，值與 API BackpackMainType 英文值完全一致（無需映射）： Heal / Ammo / Interference / Invisible / BackupEquipment / MovePointAdd / Flow。
腳本直接將 API 值寫入 type 欄位，管理員工具可依此 enum 驗證。
決策二：新增 AssemblableArmorType enum（正向邏輯）
API AssemblableAirmenType 以「可裝備的機甲類型」正向描述限制， 與現有 MechRestriction（武器用，負向邏輯）語義相反。 新建 AssemblableArmorType enum（All / Light / Medium / Heavy）， 舊 mechRestriction 欄位標記 deprecated 並保留，待後續決定是否統一。
決策三：共用 WeaponRarity enum
背包稀有度與武器稀有度的分級對應完全相同（SSSR/UR/SSR/SR/R → SS/S+/S/A/B）， 直接共用 WeaponRarity enum，不另建 BackpackRarity。 腳本以 quality 欄位對應，映射規則與 scrape-weapons.js 相同。
決策四：skill 欄位改名為 mainSkill，補入 API 欄位
舊 skill 欄位改名為 mainSkill，並新增 API 可自動填入的欄位： id（ID）、icon（SkillIcon）、description（SpecificEffects，清洗後）、 buffIds（BufCarried，split）。
分析性數值（dmg / crit / critDmg / acc / specialEffects）保留為管理員手動欄位。
mainSkill 設為選填（?），僅 SSSR 品質背包才寫入； 腳本保護既有手動填入欄位（不覆蓋 prev.mainSkill 的 dmg/crit 等）。
決策五：腳本架構對齊 scrape-weapons.js
採用與 scrape-weapons.js 相同的設計模式： --probe、--dry-run、--limit、--rarity、--auto、--no-images 等參數； list 端點取所有 ID → 逐一呼叫 detail 端點；s2t() 簡→繁轉換；interactive confirm 機制。

Script
腳本規格（scripts/scrape-backpacks.js）
// 使用方式（對齊 scrape-weapons.js 慣例）： // node scripts/scrape-backpacks.js --probe --limit=3 ← 印出原始 API 資料 // node scripts/scrape-backpacks.js --dry-run --limit=3 ← 預覽解析結果，不寫入 // node scripts/scrape-backpacks.js --limit=3 ← 實際寫入前 3 筆 // node scripts/scrape-backpacks.js ← 全量更新（互動確認） // node scripts/scrape-backpacks.js --rarity=SS ← 只處理 SS 背包（SSSR） // node scripts/scrape-backpacks.js --auto ← 略過確認直接寫入 const RARITY_MAP = { 'SSSR': 'SS', 'UR': 'S+', 'SSR': 'S', 'SR': 'A', 'R': 'B', }; const ASSEMBLABLE_MAP = { 'Light': 'Light', 'Medium': 'Medium', 'Heavy': 'Heavy', }; function parseBackpack(raw, prev) { const rarity = RARITY_MAP[raw.quality] ?? '__REVIEW__' + raw.quality; const assemblableArmorType = ASSEMBLABLE_MAP[raw.AssemblableAirmenType] ?? 'All'; const repairAmount = parseInt(raw.AmountOfRepair ?? '0', 10) || 0; const doc = { id: raw.ID, name: s2t(raw.name), type: raw.BackpackMainType, // BackpackType enum rarity, // WeaponRarity weight: parseInt(raw.weight, 10), assemblableArmorType, repairAmount, }; // 只有 SSSR 才有 WithPassiveSkills const skills = raw.WithPassiveSkills ?? []; if (raw.quality === 'SSSR' && skills.length > 0) { const sk = skills[0]; const prevSkill = prev?.mainSkill ?? {}; doc.mainSkill = { id: sk.ID, name: s2t(sk.name), icon: sk.SkillIcon ?? sk.icon ?? '', description: stripRichText(s2t(sk.SpecificEffects ?? '')), buffIds: (sk.BufCarried ?? '').split('/').filter(Boolean), // 保留管理員手動填入的數值欄位 ...(prevSkill.dmg != null && { dmg: prevSkill.dmg }), ...(prevSkill.crit != null && { crit: prevSkill.crit }), ...(prevSkill.critDmg != null && { critDmg: prevSkill.critDmg }), ...(prevSkill.acc != null && { acc: prevSkill.acc }), ...(prevSkill.specialEffects != null && { specialEffects: prevSkill.specialEffects }), }; } return doc; }
