Problem
問題背景
問題一：武器資料量增長，列表頁無法完整展示
PLAN-003 重構後，武器 Schema 包含：基礎屬性、三種射程欄位、技能列表（含 effects[]）、固定改裝、浮動改裝、元件插槽。 若再加上 PLAN-005 引入的專武天賦強化（enhancedTalentDescription）， 全部資訊在列表行展開顯示將導致嚴重的版面擁擠與閱讀困難。
問題二：專武天賦強化 DiffHighlight 需要充足的版面空間
PLAN-005 實作的專武天賦強化 DiffHighlight 區塊（黃色）包含原始天賦文字與強化後版本的差異對比， 文字量不小，在列表頁展開無法給予適當的閱讀空間與視覺層次。 只有在獨立詳情頁才能以卡片式佈局清晰呈現「有無專武的天賦差異」。
問題三：改裝浮動效果的可能選項多達 8–12 條
浮動改裝的 possibleEffects[] 包含所有可能效果、觸發條件與數值範圍， 在列表頁無法完整顯示，使用者無法評估改裝潛力。詳情頁可提供完整的改裝效果表格。
問題四：武器圖鑑頁（/weapons）已有「待確認」備註需要解決
武器頁面規劃文件明確記錄「武器詳情是否需要獨立頁面？」為待確認項目。 隨著武器資料量與功能複雜度的增加，本計畫正式確認答案為是，並提供完整的設計規格。
Design
設計決策
決策一：新增獨立路由 /weapons/:id
在 React Router 設定中新增 /weapons/:id 路由，對應 WeaponDetailPage 元件。 武器圖鑑列表頁（WeaponsPage）的每一行武器名稱加入點擊連結，導向詳情頁。 詳情頁提供「← 返回武器圖鑑」的 breadcrumb 導航。
決策二：頁面佈局採垂直單欄堆疊
由上至下依序：Hero 區塊 → 基礎屬性面板 → 技能列表 → 改裝資訊（固定 + 浮動並排）→ 元件插槽資訊。 若 isExclusive = true，技能列表中有 enhancedTalentDescription 的技能卡片下方附加黃色天賦強化 DiffHighlight 區塊， 並在 Hero 下方顯示「專武對應機師」標籤（連結至機師詳情頁）。 此佈局與現有 PilotDetailPage 的垂直堆疊設計一致。
決策三：射程以視覺格式呈現
· manhattan / orthogonal：顯示 ${minRange}-${maxRange}（如 "2-4"），並加 rangeType 標籤說明型態
· ring：顯示 ${maxRange}+（如 "2+"），標示「方形擴散」型態
射程型態標籤（manhattan / orthogonal / ring）以小字輔助說明，不佔主要空間。
決策四：技能生效方式使用顏色標籤
· carry（攜帶即生效）→ 紫色標籤
· equip（裝備中生效）→ 青色標籤
· use（使用時生效）→ 橘色標籤
與 PilotDetailPage 技能區塊的顏色語意保持一致（主動=紅在武器無此概念，不使用）。
決策五：浮動改裝顯示所有可能效果
possibleEffects[] 以表格呈現，欄位包含：效果類型（stat）、觸發條件（condition，若無則顯示「無條件」）、數值範圍（min ~ max）。 表格上方顯示「可填槽位數（slots）」，讓使用者清楚知道能同時選幾個效果。
決策六：複用 WeaponRarityBadge 元件
已有 WeaponRarityBadge 元件（PLAN-004 產出），Hero 區塊直接複用。 新增的分類標籤（type / kind）、機甲限制（mechRestriction）、裝備部位（equipSlot） 以同樣的 badge 視覺呈現，可視情況抽出為共用的 WeaponBadges 元件。
