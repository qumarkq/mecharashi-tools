Problem
問題背景
現有 PilotDetailPage 在 1920×1080 下的 UX 缺陷，以及資訊呈現不完整的問題。
問題一：六維雷達圖佔版面過大
六維屬性卡片為全寬獨立區塊（mb-6），高度約 350px，導致 Hero 資訊、雷達圖、技能標籤頁三者無法同時出現在 1080px 視窗中，使用者一進頁面便需要滾動才能看到天賦與技能。
問題二：Hero 欄位使用空間低效
Hero 為兩欄（肖像 + 資訊），右側空間浪費。六維圖可移至 Hero 右側，節省整頁的垂直高度，讓技能標籤頁大幅往上提升。
問題三：天賦「最大強化效果」需要點擊展開
使用 <details>/<summary> 折疊，玩家必須逐一展開才能看到。天賦最大強化效果是核心資訊，應直接顯示，不需要點擊。
問題四：無專武資訊
天賦欄位旁無專屬武器面板。當機師擁有專屬武器時，玩家需自行前往武器頁查找。Weapon 型別已有 isExclusive / exclusiveFor 欄位，資料已備齊，僅缺前端呈現。
問題五：神經驅動版型與遊戲介面落差大
現行為垂直 list，遊戲內為四象限格局（α 左上、β 左下、γ1 右上、γ2 右下）。垂直 list 無法讓玩家快速對應到遊戲中的芯片填入位置，降低查詢效率。
Design
設計決策
針對上述問題的版型與互動設計方向。
決策一：Hero 改為三欄佈局
grid-cols-[160px_1fr_auto] — 左欄固定肖像（160px 寬），中欄彈性顯示名稱 / 資訊 / AP 徽章，右欄放置縮小版雷達圖（260px 寬）。此布局在 1920×1080 下，整個 Hero 區域可控制在 240px 高度以內。
決策二：移除獨立六維卡片，雷達圖嵌入 Hero 右欄
刪除原本的 <div class="bg-bg-card ... mb-6"> 全寬六維區塊。RadarChart 縮小 SVG viewBox 至 260×220，作為 Hero 右欄元件直接渲染。
決策三：天賦「最大強化效果」直接展示
移除 <details>，改為在 descriptionMax 存在且不同於 description 時，直接在描述文字下方以帶色標籤 最大強化 分隔後顯示。
決策四：天賦標籤頁分左右兩欄
天賦列表（左側 ~60% 寬）＋ 專武面板（右側 ~40% 寬，固定黏在右）。未查詢到專武時顯示「暫無專屬武器」佔位卡片。 Hook：新增 usePilotExclusiveWeapon(pilotName) — 呼叫 getWeapons() 後以 w.exclusiveFor === pilotName 過濾，回傳第一筆。
決策五：神經驅動改為 2×2 四象限 Grid
grid-cols-2 grid-rows-2，依照遊戲介面對應位置：
nd[0]（α）→ row-1 col-1（左上）
nd[1]（β）→ row-2 col-1（左下）
nd[2]（γ1）→ row-1 col-2（右上）
nd[3]（γ2）→ row-2 col-2（右下）
每格顯示區域名稱標頭、芯片槽資訊、激活階層列表（Lv / minSum / 效果文字）。

3. usePilotExclusiveWeapon Hook（useFirestore.ts）
export function usePilotExclusiveWeapon(pilotName: string | undefined): HookResult<Weapon | null> { const [data, setData] = useState<Weapon | null>(null) const [loading, setLoading] = useState(true) const [error, setError] = useState<Error | null>(null) useEffect(() => { if (!pilotName) { setLoading(false); return } setLoading(true) getWeapons() .then(weapons => { const found = weapons.find( w => w.isExclusive && w.exclusiveFor === pilotName ) setData(found ?? null) }) .catch((e: unknown) => setError(e instanceof Error ? e : new Error(String(e)))) .finally(() => setLoading(false)) }, [pilotName]) return { data, loading, error } }
