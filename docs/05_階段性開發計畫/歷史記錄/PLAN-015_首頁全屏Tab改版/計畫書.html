背景與動機

PLAN-013 完成了三個資料元件（版本濃縮表、灰燼行動參考、版本時間線），並在首頁以傳統垂直堆疊方式呈現。

本計畫改版首頁布局，使其視覺上更接近鋼嵐官方網站的風格：
  • 官網使用 fullpage.js（jQuery）實現滑鼠滾輪 Page Snap；本站改用 CSS scroll-snap（原生，零依賴）達到相同效果
  • 官網 NEWS 區塊：左側 Tab Panel（約 50% 寬），右側透出角色立繪背景；本站採相同布局，三個資料面板合為 Tab 切換
  • 官網以 html { overflow: hidden } 隱藏捲動條；本站同樣隱藏，僅首頁生效，不影響其他頁面
本計畫的邊界：僅改動首頁視覺布局與捲動行為。三個資料元件（VersionQuickTable、GrayOpsPanel、VersionTimeline）的內部邏輯與資料來源維持不變。
Reference
官網技術分析（參考依據）
官網（ma.tentree-games.com）技術棧
  • fullpage.js 3.0.9（GPL v3）+ jQuery 1.10.2：頁面捲動核心，加上 html.fp-enabled { overflow: hidden } 完全屏蔽原生捲動
  • Owl Carousel：機甲/角色輪播
  • Page 錨點：['top', 'news', 'mecha', 'hero', 'world', 'feature']
  • NEWS Tab：jQuery click handler 切換 .active class，首次切換才 AJAX 載入資料
  • NEWS Panel CSS：.news-group { width: 50~70% }，左側固定寬度，右側透出角色背景
為何不直接引入 fullpage.js？

本站技術棧為 React + Vite + Tailwind，引入 jQuery 套件不合適。
CSS scroll-snap-type: y mandatory 是 W3C 標準，現代瀏覽器全面支援，效果與 fullpage.js 高度相似（滾一格跳一頁），且無任何外部依賴。
差異：CSS scroll-snap 無法做跨頁過場動畫；但對資料工具站而言，簡潔的跳頁已足夠。

Scroll-Snap
CSS Scroll-Snap 實作方案
核心原則：Page Snap 行為僅對首頁生效，其他頁面維持正常捲動。採用「首頁路由掛載時切換 CSS class」的方式隔離。
/* index.css 新增 ─────────────────────────────────────── */ /* Page Snap Container（由 HomePage 的 main wrapper 套用 .homepage-snap） */ .homepage-snap { height: calc(100vh - 4rem); /* 扣除 header h-16 */ overflow-y: scroll; scroll-snap-type: y mandatory; scrollbar-width: none; /* Firefox */ } .homepage-snap::-webkit-scrollbar { display: none; } /* Chrome/Safari */ /* 每個 Snap Page section */ .snap-page { height: 100%; scroll-snap-align: start; scroll-snap-stop: always; /* 防止快速滾動跳過 */ flex-shrink: 0; } /* 手機版：同時扣除底部 Tab Bar（3.5rem + safe-area） */ @media (max-width: 1023px) { .homepage-snap { height: calc(100vh - 4rem - 3.5rem - env(safe-area-inset-bottom)); } }
// HomePage.tsx 結構（概略） export default function HomePage() { const { data: versions, loading, error } = usePatchVersions() return ( <div className="homepage-snap"> {/* Page 1：Hero */} <section className="snap-page relative flex items-center"> {/* 左側標題區 */} <div className="max-w-[45vw] px-8 lg:px-16"> {/* Logo、站名、功能摘要、向下提示 */} </div> {/* 右半：無元素，背景立繪自然透出 */} </section> {/* Page 2：Data Tab */} <section className="snap-page flex items-stretch"> <HomeTabPanel versions={versions} loading={loading} error={error} /> {/* 右半：無元素，背景立繪自然透出 */} </section> </div> ) }
⚠️ Layout.tsx 的 <main> 須調整

目前 Layout.tsx 的 <main className="flex-1"> 會讓子元素撐高。HomePage 的 .homepage-snap 需要 固定高度（而非撐滿），否則 Page Snap 無法正常運作。

解法：將 <main> 改為 <main className="flex-1 overflow-hidden">（加上 overflow-hidden）。由於 HomePage 自己管理捲動，其他頁面繼續正常使用 overflow: auto。
Responsive
響應式設計方針

桌面（≥ 1024px）

  • Tab Panel：max-w-[52vw] 靠左
  • 右半 48vw：完全透出背景立繪
  • Page 1 左側標題：max-w-[45vw]
  • Header 高度：4rem（扣除）
  • Footer：隱藏，精簡版移至 Page 2 左側底部

平板（768px–1023px）

  • Tab Panel：max-w-[70vw] 靠左
  • 背景立繪仍部分可見（約 30vw）
  • Page 1 標題：全寬但右側透出
  • Layout 維持 desktop 模式（有 header，無底部 tab bar）

手機（< 768px）

  • Tab Panel：全寬 w-full（無右側空間）
  • 背景立繪僅 Page 1 可見（Page 2 Panel 全覆蓋）
  • 底部 Tab Bar 高度一併扣除
  • Page Snap 仍生效（觸控滑動也可跳頁）
  • Tab Panel 內捲動：手指可在 Panel 內上下滑
觸控裝置的 Page Snap：CSS scroll-snap-type: y mandatory 對觸控裝置同樣生效，手指上下滑會觸發 Page Snap（與 PC 滾輪行為一致）。Tab Panel 內部設 overflow-y: auto，手指在 Panel 範圍內滑動時，瀏覽器會優先捲動 Panel 內容，不觸發 Page Snap，行為符合直覺。

Risks
風險與注意事項
風險 1：Timeline Tab 的高度問題

VersionTimeline 在原設計是全頁垂直展開，放入固定高度的 Tab Panel 後，需要 overflow-y: auto 在 Panel 內捲動。
需確認 VersionTimeline 的展開動畫（max-height transition）在 overflow 容器內仍能正常運作，否則需微調展開面板的 max-height 上限。
風險 2：其他頁面的背景圖

body 的背景圖（homepage_background.jpg）目前全站生效。其他頁面的元件使用半透明背景色（bg-bg-dark/10），視覺上可能出現立繪透出。
短期不處理（現有頁面已有不透明的 card/section 背景）；若之後視覺問題明顯，可在非首頁路由的 body 上套用純色背景 class 覆蓋。
風險 3：scroll-snap 與觸控的邊緣情況

某些 Android 瀏覽器對 scroll-snap-stop: always 支援不穩定，快速滑動可能跳過 page。
這是可接受的 progressive enhancement 問題，不影響桌面體驗，手機降級為普通捲動也不影響功能。
回退方案：若 Page Snap 在特定裝置出現嚴重問題，只需移除 .homepage-snap 和 .snap-page class，HomPage 即回退為正常捲動的雙 section 結構，Tab Panel 仍正常運作。影響範圍極小。
