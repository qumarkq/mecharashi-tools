Problem
問題背景
現有技能欄位設計的結構性缺陷，以及為何需要重構。
問題一：平坦數值無法表達條件
PilotSkill、PilotTalent、WeaponSkill 均使用 dmg: 0, crit: 0, critDmg: 0, acc: 0 四個平坦欄位。 爬蟲腳本擷取後全部寫 0，因為這些數字只存在於文字描述中，無法自動解析。 更嚴重的是，即使人工填入，也無法表達「這個加成在什麼條件下生效」。
問題二：神經驅動數值位置錯誤
NeuralDrive（zone 層）有 dmg/crit/critDmg/acc，但正確位置應在 NeuralDriveLevel（level 層）。 每個激活閾值（2芯片 / 4芯片 / 6芯片）提供的加成各不相同，zone 層的單一數值無法表達。
問題三：無法表達技能受益對象
計算器未來需要支援隊伍計算，但目前無任何欄位標記「這個技能效果讓自己受益」或「讓隊友受益」， 計算器無從判斷哪些技能應納入隊友加成池。
問題四：Buff 無法與技能解耦
部分技能會產生 Buff（如「彈潮蓄能」、「過熱狀態」），這些 Buff 本身有數值效果。 玩家在計算器中應能獨立切換 Buff，而非只能切換產生 Buff 的技能。 目前 buffs Collection 已預留但 effects[] 尚未定義。
問題五：專屬武器強化天賦關係薄弱
WeaponSkill.enhancesTalent 只是一個字串，沒有描述「強化之後，天賦效果如何改變」。 計算器無法知道裝備專屬武器後應套用哪一組效果值。
Design
設計決策
基於上述問題，以及計算器的實際用途（Buff 面板、手動切換、隊伍加成），確定以下設計方向。
決策一：計算器定位 — Buff 面板，非回合模擬器
計算器不模擬回合流程、堆疊觸發、或複雜的狀態機。 玩家自行決定哪些技能 / Buff 為「active」，計算器加總所有 active 效果的數值輸出最終屬性面板。 因此 condition 欄位的用途是「顯示給玩家的說明標籤」，而非計算器的自動觸發判斷。
決策二：effects[] 取代平坦數值欄位
以 SkillEffect[] 陣列取代 dmg / crit / critDmg / acc。 每個 effect 條目包含 stat（影響屬性）、value（數值）、scope（受益對象）、condition（觸發條件說明）。 空陣列 [] 代表「管理員尚未填入數值」，語義清晰，不再有誤導性的 0。
決策三：機師技能繼續嵌入 pilots 文件
機師技能為機師專屬、不跨實體共用，且讀取效率優先。 繼續嵌入 pilots 文件，不遷移至 skills Collection。 skills Collection 保留給未來武器 / 模組跨實體共用的技能模板。
決策四：Buff 為獨立可切換單位
技能以 buffIds[] 關聯其產生的 Buff。 buffs Collection 中的每個 Buff 文件擁有自己的 effects: SkillEffect[]。 玩家可在計算器中獨立切換 Buff（不依賴技能開關），例如直接啟用「過熱狀態」看效果。
決策五：天賦加強版效果存在 enhancedEffects[]
PilotTalent 新增 enhancedEffects?: SkillEffect[]。 當計算器偵測到裝備了對應專屬武器時，改用 enhancedEffects 替代 effects 計算。 WeaponSkill.enhancesTalentName 指向天賦的 name 欄位以定位對應天賦。
決策六：不對複雜遊戲機制結構化
攻擊升級、PP操控、回合限制等無法被靜態計算的機制，只保留在 description 文字中。 計算器不嘗試模擬這些機制，玩家自行閱讀描述後決定是否手動加入相關 Buff。

2. 新增共用 Interface（index.ts）
/** 觸發條件（顯示給玩家的說明標籤，計算器不自動判斷） */ export interface SkillCondition { trigger: string // PilotSkillTrigger enum weaponCategory?: string // WeaponCategory：限定武器類別 hpThreshold?: number // HP% 門檻（如 50 = "HP < 50%"） minApCost?: number // 最低 AP 消耗 targetClass?: string // 目標職業（PilotClass） } /** 單一可計算效果條目 */ export interface SkillEffect { /** 影響屬性，與 Module 平坦欄位名稱對齊： * 'dmg' | 'crit' | 'critDmg' | 'acc' * 'dmg_assault' | 'dmg_melee' | 'dmg_shooting' | 'dmg_tactical' * 'dmg_blade' | 'dmg_machinegun' | ... (武器種類，同 Module) * 'range' | 'armor_rate' | 'firepower_rate' | ... (其他屬性) */ stat: string value: number scope: string // SkillScope：'self' | 'ally' | 'team' condition: SkillCondition | null // null = 此技能 active 時永遠生效 } /** buffs Collection 文件 */ export interface GameBuff { id: string // "buff_過熱狀態" name: string // "過熱狀態" description: string icon?: string buffType: string // BuffType enum maxStack?: number // 最大疊加層數（resource 類型用） duration?: number // null = 持續到消耗 / 手動關閉 effects: SkillEffect[] // Buff active 時提供的效果 }
